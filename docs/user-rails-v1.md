## 0-1. 로그인 라우팅 — 신규만 온보딩, 재방문은 대시보드 직행

로그인/인증 직후, 사용자의 **Holding(회사) 보유 여부**로 분기한다. 온보딩(J0~J5)은 "처음 한 번"이며, 이미 회사가 있는 사용자는 매번 거치지 않는다.

| 로그인 후 상태 | 목적지 |
|---|---|
| Holding 있음 (재방문) | **대시보드 직행** (온보딩 건너뜀) |
| Holding 없음 (신규/가입 직후) | **J0 모드 선택**부터 시작 |
| Holding 있고 거래 0건 | 대시보드 + 미완료 꼬리표("회사를 세웠지만 아직 투자가 없습니다 →", 4번 참조) |

- 이 분기는 Next Action 엔진 1번 규칙("설립등기 미완료 → 회사 설립")과 동일한 판단을 라우팅 레벨에서 먼저 적용하는 것.
- **회사 설립 진입점은 신규든 추가든 항상 J0(모드 선택)부터 시작한다.** 신규 가입 = 로그인 후 자동으로 J0. 기존 사용자의 "새 회사 설립하기"(회사 추가) = V1 기능이며, 이때도 J0 → J1 → J2 → J2.5 동일한 흐름을 탄다(새 회사이므로 모드를 새로 고른다). 즉 "회사명부터 묻고 모드를 빠뜨리는" 흐름은 금지.
- *(V1)* 가입 도중 이탈(모드는 골랐으나 설립 미완료)한 사용자는 다음 로그인 시 멈춘 단계부터 재개. v0에서는 단순화를 위해 미구현(테스트 단계라 중간 이탈 거의 없음).

## 1. 첫 5분 강제 여정 (아하 순간까지 끊김 없이)

**목표: 가입 → 설립등기 → 첫 매수 기록 → 내 XIRR 숫자를 처음 봄.** 이 여정은 중간에 대시보드·메뉴로 샐 수 없다. 한 줄로 이어진 단방향 레일이다.

| 단계 | 화면 | 사용자가 하는 일 (단 하나) | 버튼 문구 | 누르면 |
|---|---|---|---|---|
| J0 | 모드 선택 | 없음(고르기만) — "어떻게 시작할까요?" → **📈 챌린지로 시작**(돈 없이 오늘부터 가상 투자, 남과 경쟁 가능) 또는 **📒 장부로 시작**(과거 포함 내 기록, 나·시장과 경쟁). **라이브(연동)**는 흐리게 "준비 중"으로 노출(V3). 작은 글씨: "나중에 다른 회사도 설립할 수 있어요"(영구 선택 아님을 안내) | **[챌린지로 시작]** / **[장부로 시작]** | J1 |
| J1 | 회사명 새기기 | 회사명 입력 — 작은 칸이 아니라 화면 중앙 "간판"에 큰 글자로 실시간 새겨짐(빈칸=흐린 플레이스홀더). 카피: "워런 버핏에게 버크셔가 있었듯…" | **[회사 설립하기]** | J2 |
| J2 | 설립 등기 (한 질문씩) | 설립일(오늘 큰 버튼) → **기존 보유 종목 등기**(수량·평단 필수 + 매수일 선택, 거래 입력과 동일한 검색 시퀀스 재사용) → (선택)현금 보유분. 한 폼에 몰지 말 것. **초기 자본금은 따로 묻지 않음**(매수 평단이 곧 원금) | **[설립 등기 완료]** | J2.5 |
| J2-h | 기존 보유 종목 등기 | "지금 갖고 계신 종목이 있나요?" → **없음(스킵, 백지 시작)** 또는 **있음** → 거래 입력과 같은 행으로 보유 종목 등록(종목·수량·평단). 단독 "현재 평가액" 숫자는 받지 않는다 — 죽은 값이라 종목별 분석·시세연동이 안 됨. 등록된 종목이 기초 자본을 구성하는 자회사가 되고, 시세 연동 시 평가액이 자동 산출됨 | **[등기 완료]** 또는 **[보유 종목 없음]** | J2.5 |
| J2.5 | 설립 의식 (클라이맥스) | 없음(보기만) — 중앙에 법인 등기증서 카드(회사명·설립일·현금 보유·자회사 수·대표 이바름) + 골드 "설립 인가" 도장 스탬프 연출. "○○이 설립되었습니다 / 오늘부터 당신은 이 회사의 CEO입니다." 공유 가능한 첫 자산 | **[대시보드로 입장]** 또는 자동 진행 | J3 |
| J3 | 첫 인수 제안 | 없음 (읽기만) — "회사가 세워졌습니다. 첫 기업을 인수할까요?" | **[첫 인수 기록하기]** | J4 |
| J4 | 매수 기록 | 종목 · 수량 · 가격 · 날짜 | **[인수 체결]** | J5 — 체결 연출 후 바로 |
| J5 | 첫 성적표 | 없음 (보기만) — 대시보드로 착지, XIRR 카드가 하이라이트됨 | — | 여정 종료, 평시 모드 진입 |

- J3에서 "나중에 하기"는 **작은 텍스트 링크로만** 제공(버튼 아님). 누르면 대시보드로 가되 미완료 꼬리표가 붙는다(3번 참조).
- J5 착지 시 XIRR 카드에 1회성 강조(글로우/스크롤 포커스): "이것이 당신의 진짜 수익률입니다."
- 90일 미만이면 XIRR 대신 누적수익률 표시(xirr-spec 엣지케이스 1과 일치). 문구: "연환산 수익률은 설립 90일 후 공개됩니다."

## 1-1. 거래 입력 시퀀스 — "회계 입력"이 아니라 "딜 체결"

폼 한 장에 다 넣으면 회계 입력이고, 한 번에 하나씩 물으면 딜 체결이다. 단 무작정 쪼개 느려지면 안 된다 — 게이밍의 핵심은 단계 수가 아니라 **리듬·자동전진·즉각 피드백**이다.

`인수`(또는 매각) 버튼을 누르면 풀스크린이 카드처럼 한 질문씩 전환된다:

| 스텝 | 질문 | 입력 | 전환 |
|---|---|---|---|
| S1 | 무엇을? | 종목 검색 (토스증권식 로고+티커 행) | 탭하면 **자동 전진**(확인 버튼 없음) |
| S2 | 얼마에? | 단가 (큰 숫자 키패드) | 입력 후 전진 |
| S3 | 몇 주? | 수량 | 입력 즉시 상단에 **실시간 환산** "= ₩7,245,000 (10주 × 724,500)" |
| S4 | 언제? | 날짜 (기본값 "오늘" 큼직하게) | 대부분 탭 한 번 |
| S5 | 체결 | "삼성전자 10주 인수" 요약 + 큰 [인수 체결] | 누르면 도장/체크 연출 + "체결되었습니다" + "회사 연혁에 기록됨" |

**게이밍을 만드는 디테일 (단계보다 중요):**
- **진행 점** ●●●○○ 상단 — 게임 스테이지 느낌
- **자동 전진** — 선택 시 "다음" 없이 넘어감. 마찰 제거가 곧 쾌감
- **실시간 환산** — 수량 넣으면 총액이 살아 움직임
- **체결 연출** — 짧은 도장/체크 애니메이션(+햅틱). 과하지 않게

**안전장치 — 빠른 입력 토글.** 파워유저에겐 매번 5스텝이 마찰이다. 평소 단계형(게이밍), 길게 누르거나 토글로 한 장 폼(빠른 입력). 둘 다 같은 Event를 만든다(데이터 동일). 배당·증자·인출은 단계가 더 적다(종목 없는 현금흐름은 S1 생략).

**거래일 규칙(모드별, 5-2와 연동):** **챌린지(challenge)·라이브(live)는 S4를 건너뛰고 거래일=오늘로 고정**(소급 불가 → 순위 조작 차단). **장부(ledger)만 S4에서 과거 날짜 선택 허용**(자기 기록이라 남과 안 섞임). 즉 S4는 ledger 모드에서만 등장한다.

## 2. Next Action 엔진 (평시 모드의 단일 버튼)

대시보드 최상단에 **Next Action 버튼이 항상 정확히 1개** 존재한다. 사용자 상태를 위에서부터 검사해 처음 맞는 규칙 하나만 표시한다.

| 우선순위 | 상태 조건 | 버튼 문구 | 누르면 |
|---|---|---|---|
| 1 | 설립등기 미완료 | **[회사 설립하기]** | J1 |
| 2 | 등기 완료 & 이벤트 0건 | **[첫 인수 기록하기]** | J4 |
| 3 | 이번 분기 결산 미확인 (분기 종료 후) | **[N분기 결산 보기]** | 결산 화면(→CFO 리포트) |
| 4 | 마지막 이벤트 후 30일 경과 | **[장부 업데이트하기]** | 이벤트 기록 화면 |
| 5 | 해당 없음 (평시) | **[거래 기록하기]** | 이벤트 기록 화면 |

- 버튼은 단수다. 2개 이상 띄우지 않는다. 조건이 겹치면 우선순위 낮은 건 무시.
- 구현: `getNextAction(userState)` 단일 함수로. 화면들이 각자 판단하지 않고 이 함수만 호출.

## 3. 빈 화면 = 명령 (Empty States)

빈 상태는 빈 표를 보여주지 않는다. 화면 한가운데 한 문장 + 버튼 하나.

| 화면 | 빈 상태 문구 | 버튼 |
|---|---|---|
| 대시보드(이벤트 0건) | "아직 자회사가 없습니다. 첫 기업을 인수하세요." | **[첫 인수 기록하기]** |
| 회사 연혁 | "회사의 역사는 첫 거래에서 시작됩니다." | **[거래 기록하기]** |
| 종목 구성 | "포트폴리오가 비어 있습니다." | **[첫 인수 기록하기]** |

## 4. 미완료 꼬리표 (도망간 사용자 회수)

- J3에서 "나중에 하기"로 샌 사용자: 대시보드 최상단에 가는 띠(배너) — "회사를 세웠지만 아직 투자가 없습니다 →" 클릭 시 J4. 첫 이벤트 기록 전까지 사라지지 않는다.
- 띠는 동시에 1개만. Next Action 버튼과 같은 상태를 가리키므로 문구만 다르고 목적지는 같다.

## 5. 재방문 트리거 (능동 + 수동)

**능동:** 거래할 때 직접 옴 — Next Action 5번이 받는다.

**수동 (v0.5+ / 토스 시세 연동 후):** 사용자가 아무것도 안 해도 회사 가치는 변한다. 일일 푸시/알림 — "오늘 바름캐피탈의 가치가 +1.2% 변동했습니다." CEO가 자기 회사 주가를 매일 확인하듯, 거래 없는 날에도 들어올 이유를 만든다. **MVP에서는 구현하지 않고 STEP 6(시세 연동)과 함께 추가한다.**

**주기 (분기):** 분기 종료 시 Next Action 3번이 자동 점화 → 결산 → CFO 리포트. 이게 가장 큰 재방문 의식이다.

## 5-1. 화면 지도 (Sitemap) — 모든 "누를 곳 → 목적지"

명세에 없는 연결은 Claude Code가 임의로 처리(=죽은 버튼이 되기 쉬움)하므로, 모든 탭 가능한 요소의 목적지를 못박는다. v0 범위 기준(fast-follow 화면은 표시).

### 화면 목록
- **온보딩**: J1 회사명 → J2 설립등기(설립일·보유종목[평단 필수·매수일 선택]·현금보유, 초기자본금 안 물음) → J2.5 설립의식 → J3 첫인수제안
- **대시보드(홈)**: 메인 콘솔
- **거래 입력**: 5종 이벤트 단계형 시퀀스(S1~S5)
- **보유 종목 전체**: 포트폴리오 목록
- **종목 상세**: 단일 종목(보유·거래내역·등락 + 기본지표 PER·ROE 등, 보유 점검용 — 매수 추천 금지) + **펀더멘털 플래그**(그 회사의 "질문" 신호, V2~V3). **ETF면 ETF 투시**(L1 추종 국가/섹터 밸류에이션 V2 / L2 구성종목 가중 V3) — 기업이든 ETF든 같은 "펀더멘털 카드"
- ~~*(V1~V2)* **거장 포트폴리오**: 부자들의 포트폴리오(버핏·달리오·우드 등) — 13F 기반 보유·비중 도넛 + 포트폴리오/매수주식/매도주식 탭 + 내 구성과 비교~~ **[2026-06-27 제거됨 — 재도입 금지. spec/018 참조]**
- *(V1)* **배당 캘린더**: 월별 달력 + 보유 종목 배당락일·예상 배당금
- *(V1)* **백분위 공유 카드**: 칭호 + 분포 히스토그램 + 공유(원금 구간별 상위 %)
- **마찰비용 상세**: 매매수수료 + 세금 + ETF 총보수(TER)를 종목별로 펼침, "수수료 드래그 %p"
- **자산배분(비중) 상세**: 도넛 차트 + 항목별 비중 목록(토스/도미노식). **분류 탭 — 종목별 / 국가별 / 산업별 / 계좌별**(V0~V1 핵심 4탭). 도미노(종목/유형/계좌)보다 **국가별·산업별이 추가**된 게 차별점 — 가치투자자는 "어느 나라·어느 산업에 쏠렸나"를 종목보다 중요하게 본다. 종목 마스터의 국가·산업 태그로 **자동 분류**(사용자가 안 나눔). **ETF의 진짜 노출이 국가/산업 분류에서 드러남**(종목별론 "KODEX 미국나스닥 1개"지만 국가별론 "미국 50%"). 국가별 비중은 §8-2 ETF 국가 밸류에이션과 연결(비싼 시장에 쏠렸나). 확장: **유형별**(주식/ETF/채권/현금, V1~V2) · **스타일별**(가치/성장/배당, 칭호 연결, V1~V2) · **투시 기여별**(평가액 비중이 아니라 *실적 기여* — "비중 37%인데 순이익 기여 60%", V2)
- **투시 재무제표 상세** *(V2)*: 2단 구조. **(위) 전체 사업부 합산** — 내 지주회사 3종 재무제표: P&L(매출·영업이익·순이익) + B/S(자산·부채·자본) + 현금흐름표(영업·투자·재무 + 잉여현금흐름 FCF) + 감가상각·CapEx 표시 + 가중 ROE·ROA + 펀더멘털 플래그 요약. 각 지표는 숫자 나열이 아니라 시장 대비 비교막대·추세 스파크라인으로 시각화. **(아래) 기업별 기여 목록** — 종목별 "내 몫"(순이익·ROE 등)을 토스증권식 행으로, 어느 자회사가 실적을 끌고 가나/까먹나 한눈에. 각 행 누르면 → 종목 상세(개별 단독 재무). 기준 분기 표기
- **내 사업부 IR** *(V2)*: 보유 종목 DART 공시 피드(통합, 시간순). 공시 유형별 — 수시공시(합병·증자 등, 핵심) > 정기공시 > 지분공시. 제목+요약+DART 원본 링크(원문 가공 안 함). 펀더멘털 플래그가 해석 힌트. **진입점: 하단 탭(연혁 계열) + 대시보드 알림 배너("내 사업부 소식 ●N", 소식 있을 때만) + 수시공시 푸시.** 종목 상세에도 그 회사 공시만 필터 노출
- **벤치마크 상세**: 지수 그룹(S&P500·나스닥·코스피, 실시간) + 레전드 그룹(버크셔·국민연금 등, 공개기준) 전체 비교. 내 XIRR이 각각을 몇 %p 이겼는지
- **순자산 상세**: 자산 분류(투자·현금·연금·부동산) − 부채. XIRR(투자 실력)과는 다른 숫자임을 구분 표시. (자산분류·부채·부동산 통합은 V1)
- **활동 전체**: 이벤트 피드 목록
- **회사 연혁 전체**: 타임라인
- **설정**
- *(fast-follow / V1)* 리그·순위 화면 — **모드별 랭킹 페이지(탭 전환, 섞지 않음):** 챌린지 랭킹(🟢, 가상끼리 공평) / 라이브 랭킹(🔵, V3, 연동 검증) / 장부는 랭킹 없음(⚪, 자기·시장 비교만). 내 백분위 + 상위권 + 검증 배지. 유저 수 임계치 도달 시 활성화 · 멀티 회사 전환("내 투자회사들")
- *(fast-follow)* 분기 결산 / CFO 리포트 / 분석·비교 카드 상세
- *(V1)* **연차보고서**(설립 1년 후 발행, 카드/PDF, SNS 공유) · **투자 스타일 칭호**(다축 — 가치/성장·기술/배당/분산/방어, 등급 아닌 색깔) · **결산 스트릭**(분기 연속 회고 🔥)

### 멀티 회사 전환 (V1) — "활성 회사" 모델

여러 지주회사를 운영하고 그 사이를 오가는 메커니즘. **핵심은 UI가 아니라 "활성 회사" 배선**이다.

**구현 상태(2026-06-19):** `getPortfolio()`와 `getPrimaryHolding()`은 `active_holding` 쿠키를 읽는 `getActiveHolding()`으로 수렴했다. 쿠키가 없거나 본인 소유 회사가 아니면 가장 오래된 회사로 안전하게 폴백한다. 거래·리밸런싱·관심종목·부채·배당 동기화 등 직접 holding을 고르던 서버 액션도 활성 회사 기준으로 일원화했다.

**전환 메커니즘:**
- **`active_holding` 쿠키** 도입. 위 두 관문을 "쿠키 있으면 그 회사(본인 소유 검증), 없으면 가장 오래된 회사 폴백"으로 변경. **단일 관문이라 두 곳만 고치면 대시보드·자산·연혁·리포트·회사정보·지배구조도가 전부 활성 회사로 갱신**(개별 페이지 수정 0). 지배구조도 루트도 활성 회사라 그 위에 전환만 얹으면 됨(재작업 0).
- **직접 쿼리 audit(작업량의 핵심):** helper를 안 거치고 holdings를 직접 고르는 곳도 쿠키 기준으로 일원화해야 한다. 발견된 지점 — `dismissHomeSignal`(`app/dashboard/signalActions.ts`), `/activity` 페이지. (배선 > UI 가 실제 작업량.)

**전환 UX(구현됨):** 헤더 회사명 → `/company`(회사 정보, §145 자리)에서 "내 투자회사들" 목록을 펼친다:
- 회사 행(회사명·설립일·평가액) 탭 → 서버액션이 `active_holding` 쿠키 세팅 + `revalidatePath` → 전 화면 갱신. 현재 활성은 체크 표시.
- "+ 새 회사 설립" → 설립 플로우(J0→J2.5, §0-1) 실행 → 새 holding 생성 후 자동 활성화.
- 헤더가 활성 회사명을 표시 → 탭 → `/company`에서 전환, 흐름이 자연히 닫힌다.

**모바일(480px) 주의:** 회사들을 옆으로 펼치지 않는다 — "활성 회사 1개의 구조를 보여주고 + 목록에서 전환". 지배구조도는 항상 활성 회사 한 그루.

### 뒤로가기 원칙
- 모든 상세 화면은 좌상단 ‹ 로 직전 화면 복귀.
- 거래 입력·온보딩은 단계 내 ‹ 가 *이전 스텝*으로(화면 이탈 아님). 첫 스텝에서 ‹ 면 화면 종료.
- 상세→상세 이동(예: 활동 행 → 종목 상세)도 스택으로 쌓여 ‹ 로 순서대로 복귀.

### v0에서 "죽은 버튼" 방지 규칙
- 표의 모든 목적지는 v0에 최소한의 화면이라도 존재해야 한다(빈 상태 명령형 카피로라도).
- fast-follow 목적지(CFO 리포트·분석카드 등)는 "곧 공개됩니다" 플레이스홀더 1장으로 연결 — 버튼이 아무 반응 없는 상태는 금지.

### 탭 어포던스 규칙 (누를 수 있음을 보이게)
- **누르면 더 깊은 정보가 있는 요소에만** 어포던스를 단다: 카드 우상단 작은 › (chevron), 또는 헤더의 "전체보기 ›", 행 끝의 ›. 누름 시 살짝 눌리는 반응(scale)도 포함.
- **그 자체로 끝인 표시값엔 어포던스를 달지 않는다** — 특히 히어로 순자산 숫자는 가장 큰 숫자이고 더 깊은 정보가 없으므로 누르지 않게 둔다(누르게 하면 "뭐가 더 있나" 혼란).
- 판단 기준: "누르면 *더 깊은 정보/다음 화면*이 있는가?" 예 → 어포던스 + 연결. 아니오 → 깔끔히 표시만.
- 모든 게 눌리는 것이 아니라, 누를 만한 것만 눌리고 그것이 보이는 것이 좋은 UX다.
