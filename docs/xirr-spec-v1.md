## 1. 현금흐름 정의 (가장 중요 — 여기서 다 틀린다)

XIRR은 **투자자 주머니 기준의 외부 현금흐름**만 사용한다.

| 항목 | 부호 | 이유 |
|---|---|---|
| 설립 등기 평가액 (initial_valuation, founded_at) | **음수 (−)** | 설립일에 그만큼 투자한 것으로 간주 |
| 입금 (DEPOSIT) | **음수 (−)** | 주머니에서 나간 돈 |
| 출금 (WITHDRAWAL) | **양수 (+)** | 주머니로 들어온 돈 |
| 오늘의 총 평가액 (terminal value) | **양수 (+)** | 지금 전부 회수한다고 가정 |

### ⚠️ 배당(DIVIDEND)은 XIRR 현금흐름에 넣지 않는다

우리 모델에서 배당은 **계좌 안의 현금 잔고로 들어간다** = 내부 현금흐름이다. 배당의 효과는 terminal value(현금 잔고 포함)에 이미 반영되므로, 별도 현금흐름으로 넣으면 **이중계산**된다. 매수·매도도 마찬가지로 계좌 내부 이동이라 XIRR에 직접 들어가지 않는다 — 평가액을 통해서만 반영된다.

```
flows = [
  { date: founded_at,  amount: -initial_valuation },
  ...DEPOSIT events.map(e  => ({ date: e.date, amount: -e.amount })),
  ...WITHDRAWAL events.map(e => ({ date: e.date, amount: +e.amount })),
  { date: today, amount: +current_total_valuation }   // Σ(수량×시세) + 현금잔고
]
```

**범위 주의 — XIRR은 투자 계좌만.** `current_total_valuation`은 *투자 계좌*의 (Σ 수량×시세 + 현금잔고)만 포함한다. 부동산·연금·부채는 XIRR 현금흐름에 넣지 않는다(부동산 18억이 끼면 수익률이 부동산에 압도당해 투자 실력이 안 보인다). 순자산(자산−부채 전체)은 별도 표시값이며 XIRR과 다른 숫자다(PRD "순자산 vs 투자수익률" 참조).

## 2. 수식

연환산 수익률 r은 다음 NPV를 0으로 만드는 값:

```
NPV(r) = Σᵢ  amountᵢ / (1 + r)^(tᵢ)
tᵢ = (dateᵢ − founded_at) / 365.0   // 연 단위 경과시간
```

---

## 3. 해 찾기 — Newton-Raphson + Bisection 폴백

```
function xirr(flows):
  // 검증
  if flows에 양수와 음수가 모두 존재하지 않으면 → return null  // 해 없음
  
  // 1차: Newton-Raphson
  r = 0.1                                  // 초기 추정치 10%
  for iter in 1..100:
    npv  = Σ amountᵢ / (1+r)^tᵢ
    dnpv = Σ -tᵢ × amountᵢ / (1+r)^(tᵢ+1)  // 도함수
    if |dnpv| < 1e-12: break               // 발산 방지 → bisection으로
    r_next = r - npv / dnpv
    if r_next <= -0.999: r_next = (r - 0.999) / 2   // 정의역 보호
    if |r_next - r| < 1e-9: return r_next  // 수렴
    r = r_next
  
  // 2차 폴백: Bisection (느리지만 절대 실패 안 함)
  lo, hi = -0.999, 10.0                    // 연 −99.9% ~ +1000%
  if NPV(lo) × NPV(hi) > 0: return null    // 구간 내 해 없음
  for iter in 1..200:
    mid = (lo + hi) / 2
    if |NPV(mid)| < 1e-9: return mid
    if NPV(lo) × NPV(mid) < 0: hi = mid
    else: lo = mid
  return mid
```

## 4. 엣지 케이스 (전부 처리할 것)

1. **설립 90일 미만:** 연환산 XIRR 표시 금지(2주 수익을 연환산하면 왜곡). 대신 **누적 수익률** `(terminal + Σ출금 − Σ입금 − initial_valuation) / (initial_valuation + Σ입금)` 표시하고 "연환산 수익률 공개까지 D-N"(N=90−경과일) 카운트다운 안내.
2. **이벤트 0건 (설립 등기만):** XIRR = 등기 평가액 vs 현재 평가액 2점 계산. 정상 동작해야 함.
3. **현재 평가액이 0 또는 시세 조회 실패:** XIRR 표시하지 않고 "시세 갱신 필요" 상태. **절대 0이나 이전 값으로 조용히 대체하지 않는다.**
4. **같은 날 여러 이벤트:** 합산하지 말고 개별 flow로 유지 (수학적으로 동일하지만 감사 추적 보존).
5. **미래 날짜 이벤트:** 입력 단계에서 거부.
6. **출금이 전체 자산 초과:** 입력 단계에서 거부 (현금 잔고 음수 방지).

## 5. CAGR (보조 지표)

```
CAGR = (terminal_value / (initial_valuation + Σ입금 − Σ출금))^(1/years) − 1
```
주의: CAGR은 입금 타이밍을 무시하므로 XIRR과 다를 수 있다. 대시보드 메인은 **XIRR**, CAGR은 참고용.

### 고객용 설명 (XIRR 상세 화면에 노출 — 교육이 곧 차별화)

사용자에게 두 숫자의 차이를 쉽게 풀어 보여준다. 일반 앱은 숫자만 던지지만, 우리는 "왜 이게 더 정확한가"를 가르쳐 신뢰와 브랜드(투자를 가르치는 rational.cap)를 만든다.

- **XIRR (연복리, 메인):** "입금하고 출금한 *날짜와 금액까지* 반영한 진짜 연복리 수익률. 당신의 실제 실력."
- **CAGR (단순 연복리, 참고):** "시작과 끝만 보고 계산한 연복리. 간단하지만 중간에 돈을 더 넣고 뺀 걸 모른다."
- **왜 다른가 (예시):** "작년 말에 큰돈을 넣었다면 — CAGR은 그 타이밍을 몰라 수익률을 왜곡하지만, XIRR은 정확히 반영합니다. 그래서 토스·뱅크샐러드의 단순 수익률이 아니라, 우리는 XIRR을 메인으로 보여줍니다."
- 한 줄 카피: "이것이 당신의 진짜 수익률입니다 — 돈을 언제 넣었는지까지 계산했으니까요."

## 7. 자매 엔진 — 투시 재무제표

XIRR이 "시장이 매긴 내 수익률"이라면, **투시 재무제표(`lookthrough-financials-spec-v1.md`)**는 "내가 소유한 사업체들의 실제 펀더멘털"이다. 둘은 다른 질문에 답하며 섞지 않는다. XIRR은 V0 코어, 투시 재무제표는 V2(DART 재무 연동 후). 자세한 계산은 해당 명세 참조.
