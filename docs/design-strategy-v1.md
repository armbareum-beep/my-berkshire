# ENUF — 디자인 전략 v2 (토스/뱅크샐러드식 모던)

> 이 문서가 색·폰트·레이아웃의 단일 기준이다. 스티치 프롬프트와 Claude Code 마스터 프롬프트의 디자인 토큰은 전부 여기서 파생된다. 충돌 시 이 문서가 이긴다.
>
> **v1(다크/크림·세리프·골드)은 폐기.** 그 방향은 "고급"이 아니라 "올드"였다. 모던함은 색을 더하는 게 아니라 빼는 데서 나온다.

---

## 1. 방향 (한 문장)

**토스/뱅크샐러드식 모던.** 아주 옅은 쿨그레이 배경 + 새하얀 카드 + 부드러운 그림자. 컬러는 거의 쓰지 않고, 산세리프 하나로 굵기 차이만으로 위계를 만든다. 비싸 보이는 건 색을 써서가 아니라 안 써서다.

레퍼런스: 토스뱅크 홈, 뱅크샐러드 자산 화면, 토스증권 종목 리스트.

**구현 기반 = shadcn/ui.** UI는 shadcn/ui 컴포넌트(Radix + Tailwind) 위에 아래 토큰을 CSS 변수로 주입해 만든다. 컴포넌트는 검증된 걸 쓰고 색·라운드·그림자만 우리 토스 무드로 덮는다. 단, shadcn 기본값(`--radius: 0.5rem`, 약한 그림자) 그대로 쓰면 "흔한 SaaS 대시보드"가 되므로, 라운드를 키우고(0.875~1rem) soft shadow를 더해 토스 무드로 끌어올린다.

---

## 2. 색 — 거의 무채색, 컬러는 점처럼

**베이스 (화면의 대부분)**
- 배경 `#F2F4F6` (아주 옅은 쿨그레이 — 카드를 띄우는 바탕)
- 카드 표면 `#FFFFFF` (순백) + 부드러운 그림자(`0 2px 8px rgba(0,0,0,0.04)` 정도, 테두리 대신 그림자로 분리)
- 본문 텍스트 `#191F28` (잉크 블랙, 순검정 아님)
- 보조 텍스트 `#8B95A1` (쿨그레이)
- 구분선 `#E5E8EB`

**포인트 컬러 (아주 절제 — 버튼·링크·강조에만)**
- 프라이머리 `#3182F6` (토스 블루) — 메인 액션 버튼, 링크
- 상승 `#F04452` (레드), 하락 `#3182F6` (블루) — **한국식.** 자산 변화 화살표/숫자에만 점처럼.
- 강조 배지 배경은 아주 옅은 틴트(`#3182F6` 8% 등)로, 진한 면 채움 금지.

**철칙**
- 화면 대부분은 흑·백·회색. 컬러는 "변화"와 "단 하나의 액션"에만.
- 그림자로 카드를 띄운다(테두리선 최소). 이 흰 카드 부유감이 모던함의 핵심.
- 골드·세리프·크림 금지(v1 잔재). 그라데이션 금지. 일러스트·마스코트 최소(있으면 둥글고 플랫하게).

**shadcn CSS 변수 매핑 (globals.css에 주입)**
| shadcn 변수 | 값 |
|---|---|
| `--background` | #F2F4F6 |
| `--card` | #FFFFFF |
| `--foreground` | #191F28 |
| `--muted-foreground` | #8B95A1 |
| `--border` | #E5E8EB |
| `--primary` | #3182F6 (토스블루) |
| `--radius` | 0.875rem (shadcn 기본보다 크게) |
| 상승/하락 | #F04452 / #3182F6 (커스텀 토큰, 한국식) |
| 카드 그림자 | 0 2px 8px rgba(0,0,0,0.04) (테두리 대신 그림자로 분리) |

## 3. 타이포 — 산세리프 하나로 통일, 굵기로만 위계

- **단일 패밀리: Pretendard** (한국어 산세리프 표준, 토스/뱅샐 무드). 폰트는 이거 하나. 세리프 섞지 않는다.
- 위계는 색이 아니라 **굵기와 크기**로:
  - 히어로 숫자(순자산·XIRR): 아주 크게 + Bold(700~800)
  - 섹션 제목: Semibold(600)
  - 라벨·보조: Regular(400), `#8B95A1`
- 모든 수치 tabular-nums(자리 안 흔들리게).
- 큰 숫자 ↔ 작은 회색 라벨의 대비가 리듬. (단 폰트는 같다)

---

## 4. 레이아웃 원칙

- **카드 = 둥근 모서리 크게(16~20px) + 흰 배경 + soft shadow.** 토스 홈의 그 카드.
- **여백 넉넉.** 카드 안 패딩 크게, 카드 사이 간격 넉넉.
- **숫자가 히어로.** 각 화면 최상단에 가장 중요한 단일 숫자(순자산)가 크게.
- **리스트는 좌측 아이콘 + 라벨/숫자 + 우측 액션(작은 회색 pill 버튼).** 토스 계좌 리스트 패턴.
- **차트는 절제된 라인/도넛.** 가는 선, 면적 그라데이션 없음.
- **하단 탭바**(홈·자산·기록·연혁) — 단, **평시 화면에만**(대시보드·자산·연혁 등). 온보딩·거래 입력 등 "여정 중" 화면엔 탭바 없음(이탈 방지, 레일 원칙). **한 화면 한 가지 일.**
- **모바일 전용 단일 레이아웃(PWA).** 화면은 모바일 한 벌만 만든다. 데스크탑 브라우저에서 열어도 **중앙에 모바일 폭(max-width ~480px)으로 고정 표시**하고 양옆은 여백으로 둔다(토스·카카오뱅크 PC버전 방식). **데스크탑 전용 레이아웃·사이드바는 만들지 않는다** — 모바일이 본진이고, 작업을 한 벌로 줄이며, 향후 네이티브(KMP) 전환 시 그대로 옮겨지기 때문.
- 메인 액션 버튼은 화면당 1개 위계(`#3182F6` 채움 또는 굵은 pill). 나머지는 옅은 회색 pill.

## 4-1. 종목 표시 패턴 (토스증권식) — 종목이 나오는 모든 곳 공통

종목이 등장하는 모든 화면(대시보드 보유종목, CEO 대시보드 최근 활동, 거래 입력 종목 검색, 종목 구성 카드 등)은 동일한 행(row) 패턴을 쓴다:

`[원형 로고]  종목명(Bold) / 티커(작은 회색)   ······   현재가  등락률`

- **원형 로고:** 종목 로고를 원형으로. 로고는 저장하지 않고 **티커(symbol)로 시세 API에서 실시간 조회**(토스 종목 마스터/로고 URL). 로고 없으면 종목명 첫 글자 이니셜 원형 폴백.
- **티커 표기:** 종목명 아래 또는 옆에 티커를 작은 회색으로(예: 삼성전자 / 005930, Apple / AAPL).
- **등락 색(한국식):** 상승 빨강 #F04452, 하락 파랑 #3182F6. 등락률·현재가 변화에만.
- 거래 입력의 종목 검색도 같은 행으로 결과를 보여준다(로고+티커가 보여야 잘못된 종목 선택을 막는다).
- CEO 대시보드 "최근 활동" 피드: 최근 이벤트를 이 종목 행 + 행동 라벨로("인수 · 삼성전자 10주 · 2일 전").

---

## 5. 모션

토스처럼 가볍고 빠르게. 값 변경 시 숫자 카운트업, 탭 전환 부드럽게, 버튼 누름 반응. 과한 연출 금지 — 빠르고 매끄러운 게 모던이다.

---

## 6. 카피 톤 (디자인의 일부)

- 한국어, 명령형·평서형. "제출/저장" 금지 → "기록/체결/등기/인수/증자".
- 버튼 이름과 완료 토스트는 같은 동사: [인수 체결] → "체결되었습니다".
- 빈 화면·에러는 사과하지 않고 다음 행동을 가리킨다.

---

## 7. "구림"을 막는 체크리스트

1. 배경이 옅은 회색, 카드가 흰색 + 그림자로 떠 있는가? (테두리선으로만 나눴으면 약함)
2. 폰트가 산세리프 하나이고, 위계가 굵기로만 잡혔는가? (세리프 섞였으면 실패)
3. 화면에 컬러가 거의 없고, 색은 변화·액션에만 점처럼 쓰였는가? (면으로 칠해졌으면 실패)
4. 모서리가 충분히 둥글고 여백이 넉넉한가?
5. 토스/뱅크샐러드 옆에 놓아도 같은 급으로 보이는가?

위 5개 중 하나라도 'No'면 그 화면은 다시.
