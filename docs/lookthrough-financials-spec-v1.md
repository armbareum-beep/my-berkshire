# ENUF — 투시 재무제표 계산 명세 (Look-Through Financials) v1

> 이 문서는 XIRR 명세와 함께 ENUF의 양대 계산 엔진이다. XIRR이 "시장이 매긴 내 수익률"이라면, 투시 재무제표는 **"내가 소유한 사업체들의 실제 펀더멘털"**이다. 주가가 아니라 실체로 내 지주회사를 평가한다.
>
> 구현 언어: TypeScript. **범위: V2** (DART 등 재무 데이터 연동 필요). 로직은 지금 확정해 둔다.

---

## 0. 철학 — 왜 이게 필요한가

버크셔의 가치는 "버핏이 주식 사고팔아 번 돈"이 아니라, 보유한 회사들이 벌어들이는 실제 이익(버핏의 *look-through earnings*)이다. ENUF 사용자도 자기 지주회사를 같은 방식으로 본다: **"나는 주식을 산 게 아니라 사업을 샀다."**

XIRR / 순자산 / 투시 재무제표는 서로 다른 질문에 답한다(셋을 섞지 않는다):
- **XIRR** = 시장이 매긴 내 수익률 (money-weighted)
- **순자산** = 내 지갑의 자산 − 부채 (내가 가진 것)
- **투시 재무제표** = 내가 소유한 사업체들의 자산·부채·자본·손익을 지분만큼 합산 (내 사업의 실체)

---

## 1. 핵심 환산 공식

모든 투시 계산의 뿌리는 **지분율**이다.

```
지분율(ownership_ratio) = 내 보유수량 / 회사 총발행주식수
```

이 지분율을 회사의 각 재무 항목에 곱해 "내 몫"을 구한다.

```
내_몫_항목 = 회사_재무항목 × 지분율
포트폴리오_투시_항목 = Σ (각 종목의 내_몫_항목)   // 보유 종목 전체 합산
```

예외 — **주당 지표(EPS 등)는 이미 "주당"이므로 지분율을 거치지 않고 보유수량을 직접 곱한다.**
```
내_몫_순이익 = EPS × 내 보유수량              // EPS = 순이익 / 총주식수 이므로 동치
내_몫_매출   = 회사 총매출 × 지분율            // 총액 지표는 지분율 환산
```

## 2. 투시 손익계산서 (P&L)

```
포트폴리오 투시 매출    = Σ(회사 총매출    × 지분율)
포트폴리오 투시 영업이익 = Σ(회사 총영업이익 × 지분율)
포트폴리오 투시 순이익   = Σ(EPS × 보유수량)        // 또는 Σ(회사 순이익 × 지분율)
```

## 3. 투시 재무상태표 (B/S)

```
포트폴리오 투시 자산 = Σ(회사 총자산 × 지분율)
포트폴리오 투시 부채 = Σ(회사 총부채 × 지분율)
포트폴리오 투시 자본 = Σ(회사 총자본 × 지분율)
```
(항등식 점검: 투시 자산 ≈ 투시 부채 + 투시 자본. 반올림 오차 허용)

## 4. 투시 현금흐름표 (Cash Flow Statement) — 현금은 거짓말을 못 한다

손익(P&L)은 회계 처리로 분식 여지가 있지만 현금은 진실이다(버핏: 회계이익보다 현금). 가치투자자가 가장 신뢰하는 재무제표.

```
포트폴리오 투시 영업현금흐름(OCF) = Σ(회사 영업활동현금흐름 × 지분율)
포트폴리오 투시 투자현금흐름(ICF) = Σ(회사 투자활동현금흐름 × 지분율)
포트폴리오 투시 재무현금흐름(FinCF) = Σ(회사 재무활동현금흐름 × 지분율)
포트폴리오 투시 잉여현금흐름(FCF)  = Σ((영업현금흐름 − CapEx) × 지분율)
```

**핵심 — 투시 잉여현금흐름(FCF):** 내가 소유한 사업들이 *진짜로 손에 쥐는* 현금. `영업현금흐름 − 총 CapEx`로 계산하며 **추정이 없다**(둘 다 공시값). owner earnings(레벨3, 유지보수 CapEx 추정 필요 → 미구현)와 달리 FCF는 정확히 제공 가능하다. "추정 없는 현금 기준 실적"이 필요할 때 이 숫자를 쓴다.

(4-1의 감가상각비·CapEx는 원래 현금흐름표 항목이다. 현금흐름표가 그 둘의 제자리이며, 표시용 보조 항목으로 함께 다룬다.)

## 4-1. 투시 현금흐름 보조 항목 (표시만 — 자동 합산 금지)

```
포트폴리오 투시 감가상각비(D&A) = Σ(회사 D&A   × 지분율)
포트폴리오 투시 CapEx          = Σ(회사 CapEx × 지분율)
```

⚠️ **owner earnings(순이익 + 감가상각 − CapEx)를 자동 계산하지 않는다.** 버핏의 owner earnings는 *유지보수* CapEx를 써야 하는데, 공시에는 총 CapEx만 있고 유지/성장 구분이 없다. 구분하려면 **추정**이 들어가고, 이는 "정확한 거울" 원칙(추정값을 사실처럼 제시하지 않는다)에 위배된다.

대신 **감가상각비·CapEx를 공시값 그대로 표시**하여 사용자가 직접 owner earnings를 가늠하게 한다. 답을 떠먹이지 않고 정직한 재료를 제공한다 — 이것이 가치투자의 태도이자 rational.cap 브랜드.

## 5. 파생 지표 (가중 펀더멘털)

내가 소유한 사업 전체의 효율성. 가치투자자가 진짜 보는 숫자.

```
내 가중 ROE  = 포트폴리오 투시 순이익 / 포트폴리오 투시 자본
내 가중 ROA  = 포트폴리오 투시 순이익 / 포트폴리오 투시 자산
내 가중 부채비율 = 포트폴리오 투시 부채 / 포트폴리오 투시 자본
내 투시이익률(자본투입 대비) = 포트폴리오 투시 순이익 / 내 총 투자원금
```

## 6. 정밀도 레벨 (우선주 처리 — XIRR의 간단/정밀 분리와 동일 사상)

**레벨 1 — 공시값 사용 (V2 기본 구현):**
- 회사가 공시한 EPS·재무항목을 그대로 사용. 보통주/우선주는 *각각 별도 종목*으로 다룬다(예: 삼성전자 005930과 삼성전자우 005935는 다른 종목, 각자 보유수량으로 계산).
- 총발행주식수는 해당 종류주(보통주면 보통주 수)를 사용. 단순하고 90%+ 케이스 정확.

**레벨 2 — 우선주 정밀 (향후):**
- 순이익에서 우선주 배당을 차감해 *보통주 귀속이익*을 구하고, 보통주/우선주 각각의 청구권에 맞춰 환산.
- 자본구조가 복잡한 지주회사·금융지주까지 정밀. 데이터 요구량이 커서 후순위.

**레벨 3 — owner earnings 자동계산 (미구현, 명시적 제외):**
- 순이익 + D&A − *유지보수* CapEx. 유지/성장 CapEx 구분이 공시에 없어 추정 필요 → "정확한 거울" 원칙 위배로 **구현하지 않는다.** 재료(D&A·CapEx)만 4번처럼 표시.

## 7. 엣지 케이스

1. **ETF·리츠 보유 — ETF는 "제외"가 아니라 "한 단계 더 펼친다"(ETF look-through):** ETF는 그 자체로 회사 재무제표가 없지만, *주식의 묶음*이므로 구성종목으로 펼치면 투시가 가능하다. 단 국가/섹터별로 PER이 크게 다르므로(반도체 ETF는 시장 평균이 아니라 반도체 섹터) 두 정밀도로 나눈다:
   - **L1 — 추종 지수/섹터 밸류에이션(V2, 가벼움):** ETF가 추종하는 국가·섹터·테마 지수의 PER·PBR·배당수익률·ROE로 "싼 시장/섹터냐"를 판단(예: 미국 PER 33 / 코스피 22 / 중국 14 / 반도체 섹터 PER 30). 데이터 가볍고 80% 케이스 충분. 국가/지수 밸류에이션은 ETF 자본배분의 핵심 도구(개별주=기업 투시, ETF=국가/섹터 밸류).
   - **L2 — 구성종목 펼치기(V3, 정밀, 우리 무기):** ETF 구성종목(운용사가 매일 공개)을 가져와 각 종목 재무를 가중평균 → "이 ETF의 진짜 가중 PER·ROE." 국가/섹터 평균이 아니라 *내가 이 ETF로 실제 소유한 기업들*의 펀더멘털. 기존 투시 엔진(Σ 비중×재무)을 ETF 안에서 한 번 더 돌리는 재사용. 무겁지만 "ETF까지 꿰뚫는" 완성형.
   - **사용자 화면은 동일:** 기업을 누르면 기업 투시, ETF를 누르면 ETF 투시 — 같은 모양 "펀더멘털 카드". 내부 정밀도(L1/L2)는 숨기고 화면은 단순하게.
2. **적자 기업:** 순이익이 음수면 그대로 음수로 합산(가리지 않는다). 내 투시 순이익이 줄어드는 게 정직한 반영.
3. **총발행주식수 결측:** 지분율 계산 불가 → 그 종목은 "재무 데이터 없음"으로 표시하고 합산에서 제외(0으로 처리해 조용히 왜곡하지 않는다 — XIRR 시세결측 원칙과 동일).
4. **분기 갱신 시점:** 재무는 분기 공시라 시세(실시간)와 갱신 주기가 다르다. 항상 "기준 분기"를 표기(예: "2025 3Q 기준"). 최신 분기가 안 나왔으면 직전 분기 사용 + 표기.
5. **금융주(은행·보험):** 부채의 성격이 다르다(예금·보험부채는 영업 부채). 부채비율 해석이 일반 기업과 달라, 금융주 비중이 큰 경우 가중 부채비율에 주의 문구. **현금흐름표도 금융주는 영업현금흐름의 의미가 달라**(예금 증감이 영업활동에 잡힘) FCF 해석에 주의 — v2 레벨1은 단순 합산하되 안내만. v2 레벨1에서는 단순 합산하되 안내만.
6. **해외 주식:** 통화가 다르다. 투시 재무항목은 **비율 지표(ROE·부채비율)는 통화무관이라 그대로**, 금액 합산(투시 매출 등)은 원화 환산이 필요 → 금액 합산은 동일 통화 종목끼리만, 또는 환산 후. v2 레벨1은 국내 종목 우선.
7. **무상증자·액면분할로 주식수 변동:** 총발행주식수는 항상 *현재 시점* 값을 조회(과거 고정값 캐시 금지).

## 8. 테스트 케이스 (검증용)

**케이스 A — 단일 종목 기본:**
- 보유: A사 100주. A사 총발행주식수 1,000주, 총순이익 1,000,000원, EPS 1,000원.
- 기대: 지분율 10%. 투시 순이익 = 1,000 × 100 = 100,000원 = 순이익 1,000,000 × 10% ✓ (두 방식 일치)

**케이스 B — 다종목 합산 + 비율:**
- A사: 100주, 지분율 10%, 순이익 100,000(내 몫), 자본 500,000(내 몫)
- B사: 50주, 지분율 5%, 순이익 50,000(내 몫), 자본 400,000(내 몫)
- 기대: 투시 순이익 150,000 / 투시 자본 900,000 → 가중 ROE = 16.7%. (단순 평균 ROE와 다름을 확인 — 자본 가중)

**케이스 C — ETF 투시:**
- A사 주식 + KODEX 반도체 ETF 보유.
- 기대(L1): A사는 기업 투시. ETF는 추종 섹터(반도체) 지수 PER·PBR로 "섹터가 싼가" 표시. 화면상 둘 다 "펀더멘털 카드".
- 기대(L2, V3): ETF 구성종목을 펼쳐 가중 PER·ROE 계산(엔비디아 등 실제 보유 기업 가중). 마찰비용 TER은 별도로 함께 표기.

**케이스 D — 잉여현금흐름(FCF):**
- A사: 지분율 10%, 영업현금흐름 2,000,000원, CapEx 500,000원.
- 기대: 투시 OCF = 200,000원, 투시 FCF = (2,000,000 − 500,000) × 10% = 150,000원. 추정 없이 공시값만으로 산출됨을 확인.

## 9. 데이터 소스

- **필요 항목(종목별, 분기):** 매출 · 영업이익 · 순이익(또는 EPS) · 자산 · 부채 · 자본 · 영업/투자/재무 활동현금흐름 · 감가상각비(D&A) · CapEx · **총발행주식수(종류주 구분)**.
- **국내:** DART 전자공시 오픈API (금감원, 무료·공식). 재무제표 진짜 출처.
- **기본지표(EPS·PER·총주식수) 일부:** 토스증권 API에 있으면 우선 사용. 개발 착수 시 토스 API 문서로 커버리지 확정, 부족분은 DART 폴백.
- **해외:** 별도 재무 API(예: Financial Modeling Prep). V2 후반.
- **국가/섹터 지수 밸류에이션(ETF L1):** 주요국 지수 PER·PBR·배당수익률·ROE — KRX(국내), MSCI·각국 거래소(해외). ETF 자본배분 비교 기준.
- **ETF 구성종목(ETF L2, V3):** 운용사가 매일 공개하는 ETF 보유종목·비중(PDF/CU 구성내역). 각 종목 재무는 위 소스 재사용.
- **시장 비교 기준선:** 내 투시 ROE/PER을 "시장 평균"과 비교할 때의 기준값(코스피 PER·PBR 등)도 KRX 공개 데이터.
- 시세(매일) ≠ 재무(분기)이므로 갱신 파이프라인 분리.
- **공시 처리(IR 피드, PRD §8-4):** DART 공시는 유형별로 다룬다 — 정기공시(사업보고서)는 XBRL 숫자 추출(투시 재무제표 재료), 수시공시(합병·증자·계약·소송)는 제목+요약+원본 링크로 피드에 노출, 지분공시는 참고. 원문 수백 페이지를 그대로 노출하지 않는다(목록+링크가 기본, AI 요약은 V3). 일부 수시공시는 펀더멘털 플래그(§11)로 연결(증자→희석, 소송→우발부채).

## 10. 표시 (대시보드/결산 연계)

투시 재무제표 3종(손익계산서·재무상태표·현금흐름표)은 **결산(분기) + CFO 리포트**과 결합 → "내 지주회사 3분기 재무제표 + CFO 리포트" = 버크셔 연차보고서의 개인판. 항상 기준 분기 표기. 추정값 없음(레벨1 공시값 그대로)을 신뢰의 근거로 명시.

**3단 깊이 구조 (대시보드 → 투시 상세 → 종목 상세):**

1. **대시보드 — "내 사업부 실적" 요약 카드:** 숫자 나열 대신 **ROE·PER·PBR 3지표 로테이션**(영어 표기, 3~4초 자동 전환, ●○○ 점 표시, 탭하면 수동 전환). 각 지표마다 **시장/지수 대비 비교막대** + 좋음/주의 신호:
   - ROE(수익성, 높을수록 좋음): "내 사업부 vs 시장"
   - PER(이익 대비 가격, 낮을수록 쌈): "내 사업부 vs 코스피"
   - PBR(자산 대비 가격, 낮을수록 쌈): "내 사업부 vs 코스피"
   - **방향이 반대임을 신호로 표시**(ROE는 높을수록 🟢 / PER·PBR은 낮을수록 🟢). 단 "비싸다=나쁘다" 단정 금지 — "비싼 편"까지만(좋은 기업은 비쌀 만하다). 위치: 보유 종목 근처(같은 뿌리). 누르면 투시 상세.
   - 부채비율은 제외(전체 합산에선 업종 혼재로 해석 애매). 배당수익률·영업이익률 등 나머지는 상세에서.

2. **투시 상세 — 2단(전체 → 개별):**
   - **(위) 전체 사업부 합산:** 3종 재무제표 풀세트. 단 숫자표 나열이 아니라 **시장 대비 비교막대·추세 스파크라인**으로 시각화(가치투자자는 "높나 낮나, 좋아지나 나빠지나"를 보지 소수점을 보지 않는다).
   - **(아래) 기업별 기여 목록:** 종목별 "내 몫"(순이익·ROE 등)을 토스증권식 행으로. "어느 자회사가 실적을 끌고 가나/까먹나"가 한눈에(삼성전자가 내 순이익의 60% 등) — 자본배분가의 시각. 각 행 → 종목 상세.

3. **종목 상세:** 개별 기업 단독 재무제표 + 펀더멘털 플래그(§11).

## 11. 펀더멘털 플래그 — 재무제표가 던지는 질문 (V2~V3)

투시 재무제표 위에 얹히는 해석 레이어. "숫자를 보여주기"를 넘어 **"숫자가 무엇을 의심하게 하는지"**를 안내한다. rational.cap(가르치는 브랜드)의 정점.

### 절대 원칙 — 진단이 아니라 질문

같은 신호가 좋은 뜻일 수도 나쁜 뜻일 수도 있다(매출 급증 = 떨이일 수도, 진짜 성장일 수도). 따라서 **"이 회사 위험합니다" 같은 단정을 절대 하지 않는다.** 단정은 추정이고("정확한 거울" 위배), 투자 조언이라 자문업 경계 리스크가 있다. 대신 **"플래그 + 질문 + 확인할 곳"** 형식:

- 🚩 **신호:** "영업이익(100억)과 순이익(30억) 차이가 큽니다"
- ❓ **질문:** "일회성 비용이나 영업외손실이 있었을 수 있습니다. 무엇이 순이익을 깎았는지 확인해보세요."
- 📍 **확인처:** "→ 이 회사 IR 공시 / 손익계산서 영업외항목"

똑똑한 투자자가 던질 질문을 대신 던져 사용자가 스스로 파고들게 한다. 답을 주지 않는다.

### 1급 규칙 — 이익의 진짜 여부 (핵심, V2 우선 구현)

모든 1급 규칙은 하나의 질문으로 수렴한다: **"이 이익이 진짜인가?"** 매출·이익 숫자만으론 못 잡고, 현금흐름표·매출채권·재고를 넣었기에 가능해진 검증.

1. **순이익 ↑ 인데 영업현금흐름 ↓** — 이익의 질. *가장 중요.* 이익은 느는데 현금이 안 들어오면 분식 1순위 신호.
2. **순이익 vs 영업이익 괴리 큼** — 일회성/영업외손익 점검.
3. **매출채권이 매출보다 빠르게 증가** — 밀어내기(안 팔리는데 외상으로 매출 계상). 현금 안 들어옴과 연결.
4. **재고자산이 매출보다 빠르게 증가** — 안 팔려 쌓이는 중, 떨이/평가손실 임박 가능.

### 2급 규칙 — 건전성 (V2 후반)

5. **이자보상배율 < 1** (영업이익 / 이자비용) — 영업이익으로 이자도 못 냄(좀비기업). 가장 직관적인 위험 신호.

### 확장 후보 (명세에만, 미구현)

부채비율 급증, 유상증자 반복(지분 희석), CapEx 급감(투자 위축), 배당성향 급변, 영업이익률 추세 등. 해석이 갈려 헛플래그가 많아질 수 있어(부채=레버리지냐 위험이냐) 초기엔 제외. 1급+이자보상배율 5개로 시작.

### 표시 위치

- **종목 상세 화면:** 그 회사의 플래그(개별 점검 맥락).
- **투시 재무제표 상세 + CFO 리포트:** 보유 기업 중 플래그 뜬 회사를 모아 요약. CFO 리포트이 문장으로 풀어줌("A사는 이익이 늘었으나 현금흐름이 줄어 주의가 필요합니다").

### 데이터·범위

- 데이터는 투시 재무제표와 동일 소스(DART). 추가 비용은 룰 엔진뿐.
- 모든 비교는 전년동기/전분기 대비. 기준 시점 표기.
- 범위: V2~V3 (투시 재무제표 위에 얹힘).

## 12. v1.1 추가 — 규칙기반 밸류에이션·자본배분·현금흐름 패턴 (사용자 방법론)

> 출처: 사용자가 수년간 직접 쓰던 가치평가 시트(버크셔·국내주식 검산 완료). **DCF 같은 주관 추정이 아니라 공시값 + 고정 보수 규칙**이라 "정확한 거울" 원칙과 합치. 이름은 가르치는 브랜드에 맞게 풀어쓰되 공식은 보존. 구현은 §10 투시 대시보드와 같은 데이터 레이어 위에. **순서: 배당(B) 이후 "투시 펀더멘털 + 안전마진" 마일스톤** — 수기 입력 브릿지로 시작(아래 12-6).

### 12-1. 오너이익 (Owner Earnings, 총CapEx 기준) — §4-1·§6 보완

```
오너이익 = 순이익 + 감가상각비(D&A) − 총 CapEx
```

§6 레벨3은 오너이익 자동계산을 **제외**했는데, 그 이유는 버핏의 *유지보수* CapEx 추정 문제였다. **총 CapEx는 공시값이라 추정이 없고**, 총CapEx ≥ 유지보수CapEx 이므로 오너이익을 보수적으로 **과소** 계상(안전 방향). 따라서 **"오너이익(총CapEx 기준)"으로 라벨을 명시**하면 원칙 위배가 아니다. 아래 자본배분·내재가치 지표의 공통 재료라 v1.1에서 채택.
검산(버크셔 2024, 단위 $M): 88,995 + 12,855 − 18,976 = **82,874**.

### 12-2. 자본배분 효율 지표 (경영진이 유보이익을 가치로 바꾸는가)

가치투자 = 자본배분. 회사가 이익을 배당 안 하고 쌓아둘(유보할) 때, **그 유보가 정당한가**를 두 각도로 본다 — 버핏의 "유보 1달러가 ≥1달러 가치를 만드는가" 테스트.

```
유형자본수익률(RONTE) = 오너이익 / 순유형자산           // 순유형자산 = 자본 − 무형자산. 자산 효율(영업)
이익전환력(RNI = RORE) = 순이익성장 / 잉여금성장          // 유보 1원이 "이익"을 얼마나 더 만들었나(내부 재투자 수익률)
1달러유보테스트(RMC)   = (시총·가치 성장) / (유보이익 성장)  // 유보 1원이 "시장가치"를 얼마나 만들었나(외부 검증). >100%면 통과
```
- **RNI(내부)와 RMC(외부)는 짝**: RNI = 유보가 *이익*을 만드나(영업 관점, 유보이익 재투자수익률 RORE), RMC = 유보가 *가치*를 만드나(시장 검증, 버핏 1달러 테스트). 둘 다 "이 경영진에게 계속 돈을 맡겨도 되나"의 답.
- 검산(버크셔): RONTE = 82,874 / 533,137 = **15.54%**. RNI = 7,578 / 251,592 = **3.01%**(단년이라 낮게 보임). RMC = **176.44%**(유보 1원→1.76원 가치).
- ⚠️ **단년 금지·다년 필수**: 성장(특히 순이익)은 일회성·평가손익으로 출렁(버크셔 GAAP 순이익은 미실현손익 포함) → **5년+ 평균**이라야 의미. 단년 RNI는 오해를 부른다. (그래서 데이터 무겁고 안전마진 12-3보다 후순위.)
- 이름은 사용자 약자(RONTE/RNI/RMC) 대신 위 한글로(브랜드=가르침). 약자는 부제로 병기 가능.

### 12-3. 내재가치 & 안전마진 (Margin of Safety) — 헤드라인 가치지표 ⭐

주관 DCF 없이, **오너이익을 보수적 자본환원율로 나눠** 내재가치를 구하고 가격과 비교한다.

```
자본환원율(보수) = 미국채 10년물 수익률 × 2
내재가치①        = 오너이익 / 자본환원율
내재가치②(성장반영) = 오너이익 / (자본환원율 − 성장률/10)   // 성장은 1/10로 보수 할인
안전마진(금액)   = 내재가치 − 시가총액
할인율(%)        = 안전마진 / 내재가치                      // +면 쌈(할인), −면 비쌈(할증)
```
- 검산(버크셔, 10년물 4.49%): 내재가치① = 82,874 / 8.98% = **923,490**, 내재가치②(성장 12.72%) = 82,874 / (8.98% − 1.272%) = **1,075,971**. 안전마진① = −131,174(−14.2%), ② = +21,307(+1.98%).
- **포트폴리오 게이지**: Σ내재가치 vs Σ시총 → "내 포트폴리오는 내재가치 대비 X% 할인/할증". 자본배분 규율의 핵심 신호.
- 입력: 오너이익(공시/수기), **10년물 금리(공개 API 자동)**, 성장률(다년치 또는 사용자 추정).
- 원칙: 자본환원율×2·성장률/10 은 **고정 보수 규칙**(사용자 방법론). 임의 추정이 아니라 규칙이므로 "정확한 거울" 유지. 규칙값은 lib/config 상수로(바꾸면 그 파일만).

#### 12-3a. 원리 — 이건 staged DCF를 보수적으로 붕괴시킨 것

원래 DCF는 2단: `Σ FCFₜ/(1+r)ᵗ (명시 5~10년) + 터미널가치/(1+r)ᴺ`. 연도별 예측은 **거짓 정밀**(성장·터미널 가정 조금에 답이 급변). 가치투자는 반대로 — **성장 0(또는 미미)을 깔고도 싸면 산다**, 실제 성장은 공짜 업사이드.
- 성장=0이면 staged DCF는 영구연금 `V = 오너이익/r`로 붕괴 = **이익력가치(EPV, Greenwald)** = 내재가치①.
- 성장 살짝(1단 Gordon `V = OE/(r−g')`, g'=성장/10) = 내재가치②.
- 즉 우리는 **연도별 FCF 예측을 하지 않는다**(레벨3 owner earnings 자동계산 제외와 같은 정신). EPV/보수 Gordon만.
- **"PER 10 = 염가" 직관과 동치**: 성장0 EPV의 손익분기 PER = 1/r. r=10년물×2≈9% → 손익분기 PER≈11. PER 11 아래면 성장 0 가정에서도 내재가치보다 쌈. 두 규칙(PER컷·안전마진식)은 같은 한 규칙.

#### 12-3b. 할인율(10년물×2) 근거와 한계

- r = **요구수익률**. CAPM(무위험+β×ERP)≈9~10%, 장기 주식수익≈8~10% → **10년물×2(≈9%)가 그 범위**. "×2"는 위험프리미엄+안전마진을 한 숫자에 넣은 것(주식이 채권의 2배 수익을 줄 때만 산다).
- 장점: **금리 상승→r↑→내재가치↓ 자동조정**(채권이 좋아지면 주식 기준이 빡세짐). 단순.
- 버핏과 차이: 버핏은 무위험(×1)로 할인+안전마진은 **가격**에서. 사용자는 안전마진을 **할인율(×2)**에 → 더 보수적. 둘 다 유효.
- ⚠️ **한계**: ① 위험 차등 없음(안정주·변동주 동일 ×2) — 집중·이해 종목 전제면 방어 가능. ② **저금리 극단 왜곡**: 10년물 1%→×2=2%로 분모 과소→내재가치 폭발(다 싸 보임), 8%→16%로 과빡. → **환원율 바닥(예: min 8%)** 두는 게 안전. 구현 시 lib/config 상수 + 바닥 옵션.

#### 12-3c. 프레이밍 원칙 — 판결 아니라 렌즈 (성장투자자 포용)

이 지표군은 정통 가치투자라, 성장투자자에게 "비싸다=틀렸다"는 **판결**로 보이면 이탈한다. **판결 금지, 렌즈로 제공**(§11 "진단 아니라 질문"과 동일).

- **implied growth(내재 성장률) — 같은 식을 뒤집어 중립화:** 안전마진을 "X% 할증"으로 단정하는 대신, 가격이 *가정하는* 성장률을 보여준다.
  ```
  implied g = r − (오너이익 / 시가총액) = 요구수익률 − 이익수익률
  ```
  표시 예: "시장은 이 회사가 연 **Y% 성장**한다고 가정 중. 당신 생각엔?" → 가치투자자는 "무리"로, 성장투자자는 "난 더 본다→싸다"로 쓴다. **같은 숫자, 각자 판단.** 역DCF는 성장투자자가 자기 성장 뷰를 검증하는 핵심 도구.
- **성장 지표 병행:** 데이터 레이어가 주는 매출성장·이익성장·PEG·성장 추세를 안전마진 옆에 함께(성장투자자의 ROE 격).
- **무판단 톤:** "좋은 기업은 비쌀 만하다 — '비싼 편'까지만"(§10·§11). 운용스타일 진단도 성장형을 '틀림'으로 보지 않음(다른 archetype).
- **정체성(중요):** 제품은 *rational*(합리적)이지 *dogmatic value*가 아니다. **가치=척추, 성장 포용.** 안전마진/implied growth는 열풍에 휩쓸리지 않게 잡아주는 **하이프 카운터웨이트**(성장투자자에게도 "내가 얼마의 성장을 사고 있나"를 알려주는 약). 단 데이트레이딩·모멘텀·테마추격 도구는 만들지 않음(타깃 아님).
- 코어(XIRR·자산배분·리밸런싱·외화·손익·배당)는 전략 중립 — 가치색은 이 가치평가 레이어뿐. 그래서 렌즈 프레이밍만 지키면 성장투자자도 앱 대부분을 그대로 쓴다.

### 12-4. 손익분기 배수 (BEP, 성장반영 회수연수)

겉PER은 회수기간을 과대평가한다(이익이 자라면 더 빨리 회수). 누적이익이 가격을 회수하는 실제 연수:

```
BEP = ln(1 + PER × 성장률) / ln(1 + 성장률)     // 성장률 0이면 BEP = PER
```
- 검산(버크셔): PER 12.73, 성장 12.72% → ln(1+12.73×0.1272)/ln(1.1272) = **8.04년**. "겉PER 12.7이라도 실질 손익분기는 8년."
- PER은 **오너이익 기준**(시총/오너이익) 사용 가능 — 사용자 방식. 표시: "실질 회수 N년" 보조 지표.

### 12-5. 현금흐름 부호 패턴 플래그 (§11 확장 — CFO 리포트용)

현금흐름표 3항목의 **부호 조합**이 회사의 국면을 말한다(영업/투자/재무 = OCF/ICF/FinCF).

```
정상(성숙 우량): (+, −, −)  영업으로 벌고, 투자하고, 주주환원·부채상환
성장 초기:       (+, −, +)  벌지만 더 투자하려 외부조달
경고(요주의):    (−, −, +)  영업현금 까먹는데 투자도 하고, 외부조달로 연명 → 질문 필요
재무위기 의심:   (−, +, +) 또는 (−, +, −)  자산 팔아 버팀
```
- §11 "진단 아니라 질문" 원칙 유지: 패턴별 **신호 + 질문 + 확인처**. 예: (−,−,+) → "🚩 영업현금이 마이너스인데 외부조달로 메우는 중. ❓성장 투자인지 적자 연명인지 확인. 📍영업현금흐름·증자/차입 공시."
- §11-1급 규칙(순이익↑·영업현금↓ / 영업이익·순이익 괴리)과 **묶어** CFO 리포트가 문장으로 요약("A사는 이익은 늘었으나 현금흐름 부호가 (−,−,+)로 바뀌었습니다").
- 추세(전분기/전년 부호 변화)를 함께 — 부호가 (+,−,−)→(−,−,+)로 **바뀐** 시점이 핵심 경고.

### 12-6. 데이터·시점·시퀀싱 (자연스러운 순서)

- **선행 재료**: 오너이익(순이익·D&A·CapEx), 자본·무형자산, 총주식수, 다년치(성장률), 10년물 금리.
- **수기 입력 브릿지(먼저)**: 사용자가 이미 수기로 하던 입력을 그대로 받는다 — 종목별 펀더멘털 입력 화면 → §10 투시 대시보드 + 12-3 안전마진을 **DART 없이 즉시** 가동. 10년물만 공개 API 자동. → V2(자동)를 v1(수기)로 끌어내려 배당 다음 마일스톤으로 가능.
- **자동 교체(나중)**: DART/토스 연동되면 수기 입력 자리에 자동 채움(§9). 사용자는 검토·보정만.
- **표시 깊이**: 12-3 안전마진 = 대시보드 헤드라인 게이지 / 12-2·12-4 = 투시 상세·종목 상세 / 12-5 = CFO 리포트·플래그. (§10 3단 구조에 얹음)
- BEP·자본배분 지표는 성장률(다년치) 의존 → 안전마진 다음. 12-5 현금흐름 패턴은 §11 플래그와 함께 CFO 리포트 단계.
