# Spec 001 — 디자인 폴리시 패스 (Toss/뱅크샐러드 급 일관성)

> 상태: Draft · 작성 2026-06-17 · 단일 기준: `docs/design-strategy-v1.md` (특히 §7 체크리스트)
> 방식: Spec Kit (spec → plan → tasks). 이 문서는 **무엇을·왜**. 구현 방법은 `plan.md`, 실행 목록은 `tasks.md`.

## 1. 배경 / 목표

분석 엔진(XIRR·투시·펀더멘털·섹터·자산배분·리밸런싱·배당·다자산)은 사실상 완성됐다. 다음 레버리지는 **기능이 아니라 그 깊이를 담는 그릇(디자인)**이다. 창업자가 직접 도그푸딩하며 유저를 받는 전략이므로, 매일 보는 화면이 "토스 옆에 둬도 같은 급"이어야 하고, 곧 만들 SNS 공유 카드(획득 채널)도 이 기준 위에 얹혀야 재작업이 없다.

설계도가 "카드 종류 안정 후 디자인 폴리시 패스 → 그 다음 카드 개인화"로 미뤄둔 그 시점이다.

**한 줄 목표:** 전 화면이 `design-strategy-v1.md §7` 5개 체크리스트를 통과하고, 공용 컴포넌트 한 벌로 일관성이 *구조적으로* 강제되게 한다.

## 2. 현재 상태 (감사 근거)

29개 화면 + 공용 컴포넌트를 §7·토큰 기준으로 전수 감사함(2026-06-17, 4-에이전트 병렬). 토큰 레이어(`globals.css`)는 spec과 정확히 일치하나, **컴포넌트 레이어가 비거나 깨져** 화면마다 카드·행·색을 문자열로 재구현 중. 9개 테마로 수렴(상세·파일·라인은 `tasks.md`).

## 3. 범위

**대상:** `src/app/**/page.tsx` 29개, `src/components/**`, `src/app/globals.css`, `src/app/layout.tsx`.

**비범위 (이번에 안 함):**
- 기능·로직 변경, 새 기능(SNS 공유 카드·결산 스트릭 등은 이 폴리시 *이후*).
- 다크모드(앱은 라이트 단일), 데스크탑 전용 레이아웃(모바일 단일 유지).
- 카드 개인화(드래그 재배치) — 폴리시 *다음* 단계(설계도 순서).
- IA 재설계(카드→전용페이지 model A 유지).

## 4. 요구사항 (Acceptance 포함)

각 요구사항은 §7 통과를 위한 것. 검수는 화면별 §7 5점 + 아래 조건.

- **R1 (T1) 단일 카드 프리미티브.** 모든 카드 표면이 하나의 컴포넌트(그림자 분리, `rounded-2xl`, soft shadow)를 통과한다. 수동 `bg-card shadow-card` 문자열 복제 0건. `border/ring`으로 분리한 카드 0건.
  - *Accept:* `grep "rounded-2xl bg-card" src/app src/components` → 직접 사용처가 공용 컴포넌트 내부로 수렴. `ring-1`로 분리된 카드 없음.
- **R2 (T2) 히어로 카운트업.** 모든 히어로 숫자(순자산·평가액·누적손익·투시순이익·규율점수·분기수익률·배당·총현금)가 마운트 시 카운트업(`prefers-reduced-motion` 존중, `tabular-nums` 유지).
  - *Accept:* 공용 `CountUp` 1개가 모든 히어로에 적용. reduced-motion이면 즉시 표시.
- **R3 (T3) 색 시맨틱 고정.** `--rise`/`--fall`(빨강/파랑)은 **시세 등락에만**. 경고·위험·주의 = `--warn`(앰버)/`--warn-tint`. 통과/중립 상태 = 잉크/회색. FinancialHealth ✓/✗ 역전 수정.
  - *Accept:* 비-시세 맥락에서 `--rise`/`--fall`(또는 `var(--fall)` 인라인) 사용 0건. 경고는 전부 `--warn`.
- **R4 (T4) 무채색 원칙.** 카드 표면은 흰색. 솔리드 `bg-primary` 채움은 화면당 **메인 액션 1개**에만. 상태·네비 칩은 중립 세그먼트(`bg-card shadow-sm`, `/cash` 탭 패턴 재사용). 도넛은 토널/무채+포인트.
  - *Accept:* 화면당 솔리드 primary 면 ≤1. 전면 틴트로 칠한 카드 표면 0건. 도넛 팔레트 재정의.
- **R5 (T5) 차트 라인 원칙.** 면적 그라데이션 0건. 선=가는 단색, 채움은 평면 저투명 또는 없음.
  - *Accept:* `linearGradient`/`<Area fill=url(...)>` 0건.
- **R6 (T6) 단일 종목행.** 종목이 나오는 모든 곳이 하나의 `StockRow`(원형 로고/이니셜 + 종목명 굵게 + 티커 회색 + 등락색)를 쓴다. 검색·매수 결과행엔 현재가·등락 포함.
  - *Accept:* 대시보드·수익률·배당·연혁·자산배분·검색이 동일 행 컴포넌트. 이모지 원형·무아바타 행 0건.
- **R7 (T7) 탭바 위치.** `BottomTabBar`는 평시(대시보드·자산·연혁·검색 등) 화면에만. 거래 입력·온보딩·인증 플로우엔 없음. BottomTabBar 주석을 spec과 일치시킨다.
  - *Accept:* `/transactions`·`/acquire`·온보딩·로그인에 탭바 없음.
- **R8 (T8) 카피·빈화면·확인.** 빈/에러 화면은 사과 없이 다음 행동을 가리킨다(CTA 링크). 버튼 동사=토스트 동사, "제출/저장" 금지. 삭제 확인은 인앱(네이티브 `window.confirm` 제거).
  - *Accept:* 감사된 빈 상태 전부 CTA 보유. "저장" 버튼 0건. `window.confirm` 0건.
- **R9 (T9) 토큰 위생.** 죽은 `dark:` 클래스 제거(라이트 단일), 색은 `text-rise/fall/warn` 유틸 사용, 도넛 인터랙션 터치 대응, 아이콘셋 일원화(lucide).
  - *Accept:* 변경 파일 lint 클린, 인라인 색 style 최소화, 도넛 onClick/touch 동작.

## 5. 검수 방법

- `npx tsc --noEmit` · `npx eslint` 클린(변경 파일).
- §7 5점 체크리스트를 핵심 화면(대시보드·자산·종목상세·자산배분·수익률)에서 통과(스크린샷 확인 — `run`/`verify` 스킬).
- 색 시맨틱 grep 게이트(R3/R4/R5 accept 조건).
- 기능 회귀 없음(렌더·계산 불변).

## 6. 성공의 모습

토스/뱅샐 옆에 핵심 화면을 놓아도 같은 급. 빨강/파랑은 오직 시세에서만 뛰고, 화면은 흰 카드가 그림자로 떠 있는 무채색. 히어로 숫자가 부드럽게 차오르고, 종목은 어디서나 같은 얼굴. 이 기준 위에서 다음 단계(SNS 공유 카드·결산 스트릭·카드 개인화)가 재작업 없이 얹힌다.
