# Feature Specification: 간편모드 UI 업그레이드 (브랜드 마크 + 행 진입 어포던스)

**Feature Branch**: `006-simple-mode-ui`
**Created**: 2026-06-23
**Status**: Draft
**Input**: User description: "간편모드 UI 업그레이드 — 헤더 브랜드 마크(ENUF 워드마크)와 보유종목 리스트 행 상세진입 화살표(›) 어포던스 추가. 키움 간편모드 UI를 참고하되 ENUF의 토스식 디자인 토큰을 따른다."

**Clarifications**: 브랜드 마크는 별도 심볼/아이콘 없이 "ENUF" 글자(워드마크)만 사용한다(사용자 확정, 2026-06-23).

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 헤더 브랜드 마크로 앱 정체성 확립 (Priority: P1)

사용자가 앱을 열면 화면 좌상단에서 ENUF 워드마크(글자 로고)를 즉시 본다. 키움이 좌상단 '키움자산'으로 한눈에 어떤 앱인지 알리듯, ENUF도 자기 이름을 또렷한 브랜드 타이포로 보여준다. 현재는 그 자리에 회사명 텍스트만 있어 "내 지주회사 운영 콘솔"이라는 정체성이 약하다.

**Why this priority**: 사용자가 비교 시 가장 먼저, 그리고 직접 짚은 차이다("로고도 그렇고"). 브랜드 마크는 모든 화면에서 반복 노출되어 앱의 신뢰감과 정체성을 좌우하며, 다른 개선보다 체감 효과가 크다.

**Independent Test**: 헤더에 브랜드 마크만 추가해도 독립적으로 가치가 있다. 대시보드 홈을 열어 좌상단에 ENUF 마크가 보이고, 마크와 회사명이 시각적으로 구분되는지 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 대시보드 홈에 있을 때, **When** 화면을 보면, **Then** 좌상단에 ENUF 워드마크가 표시된다.
2. **Given** 워드마크가 표시된 상태에서, **When** 회사명·설립일 등 기존 헤더 정보를 함께 보면, **Then** 워드마크와 회사 정보가 시각적으로 충돌 없이 구분되어 읽힌다(워드마크가 회사명보다 상위 브랜드로 읽힌다).
3. **Given** 사용자가 작은 모바일 화면(폭 360px)을 쓸 때, **When** 헤더를 보면, **Then** 워드마크와 우측 액션 버튼이 잘리거나 겹치지 않는다.

---

### User Story 2 - 보유종목 행에서 상세 진입 어포던스 (Priority: P2)

사용자가 보유종목 목록에서 한 종목을 누르면 상세로 들어갈 수 있는데, 지금은 "누를 수 있다"는 신호가 약하다. 키움처럼 각 행 우측 끝에 진입 화살표(›)를 두어, 행을 탭하면 상세 시트로 이동한다는 것을 명확히 알린다.

**Why this priority**: 정체성(P1)이 자리 잡은 뒤의 사용성 개선이다. 상세 바텀시트는 이미 존재하므로(005), 진입 신호만 보강하면 발견율이 올라간다. 단독으로도 가치가 있으나 브랜드 마크보다 체감 우선순위가 낮다.

**Independent Test**: 보유종목 목록의 각 행 우측에 화살표를 추가하고, 행을 탭하면 해당 종목 상세 시트가 열리는지 확인하면 독립 검증된다.

**Acceptance Scenarios**:

1. **Given** 보유종목 목록이 표시될 때, **When** 각 종목 행을 보면, **Then** 행 우측 끝에 상세 진입을 뜻하는 화살표(›)가 표시된다.
2. **Given** 종목 행에 화살표가 있을 때, **When** 사용자가 행 영역을 탭하면, **Then** 해당 종목의 상세 시트가 열린다.
3. **Given** 평가금액·수익률 등 우측 정렬된 숫자가 있을 때, **When** 화살표가 추가되어도, **Then** 숫자와 화살표가 겹치지 않고 금액 가독성이 유지된다.

---

### Edge Cases

- 종목명이 매우 길 때(예: "KODEX 코리아밸류업"): 이름은 말줄임(…)으로 처리되고 브랜드 마크/화살표 영역을 침범하지 않는다.
- 보유종목이 0개일 때: 목록이 비어 있어도 헤더 브랜드 마크는 정상 표시된다.
- 회사명이 길 때: 워드마크와 회사명이 좁은 화면에서 줄바꿈/말줄임으로 안전하게 처리된다.
- 종목 행이 상세 시트를 지원하지 않는 항목(예: 현금)일 때: 화살표 노출 여부가 진입 가능 여부와 일치한다(진입 불가 항목엔 화살표를 두지 않는다).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 대시보드 홈 헤더 좌상단에 ENUF 워드마크(글자 로고, 별도 심볼/아이콘 없음)를 표시해야 한다.
- **FR-002**: 워드마크는 기존 회사명·설립일·모드 정보와 시각적으로 구분되어 함께 표시되어야 한다(워드마크가 상위 브랜드, 회사명이 하위 정보로 위계가 읽혀야 한다).
- **FR-003**: 워드마크는 ENUF의 기존 디자인 토큰(색·모서리·그림자·폰트)을 따라야 하며, 새로운 시각 언어를 도입하지 않아야 한다.
- **FR-004**: 시스템은 보유종목 목록의 각 행 우측 끝에 상세 진입을 의미하는 화살표(›) 어포던스를 표시해야 한다.
- **FR-005**: 사용자가 화살표가 있는 종목 행을 탭하면 시스템은 해당 종목의 상세 시트를 열어야 한다.
- **FR-006**: 화살표 어포던스는 상세 진입이 가능한 항목에만 표시되어야 하며, 진입 불가 항목(예: 시트가 없는 항목)에는 표시되지 않아야 한다.
- **FR-007**: 브랜드 마크와 행 화살표 추가는 기존 평가금액·수익률·정렬·계좌별 접이식 등 기존 정보의 가독성과 레이아웃을 훼손하지 않아야 한다.
- **FR-008**: 워드마크는 별도의 그림 심볼/아이콘을 포함하지 않고 "ENUF" 글자만으로 구성되어야 한다.

### Key Entities *(include if feature involves data)*

본 기능은 데이터 모델 변경 없이 표현(presentation) 계층만 다룬다. 신규 엔티티 없음.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 대시보드 홈을 연 사용자가 1초 안에 좌상단에서 ENUF 워드마크를 식별할 수 있다.
- **SC-002**: 폭 360px~480px 범위의 모든 모바일 폭에서 헤더 요소(워드마크·회사명·액션 버튼)가 잘림/겹침 없이 표시된다.
- **SC-003**: 보유종목 목록의 모든 진입 가능 행에 화살표가 표시되고, 화살표가 있는 행 탭 시 100% 상세 시트가 열린다.
- **SC-004**: 변경 후에도 평가금액·수익률 숫자가 기존과 동일하게 잘림 없이 읽히며, 기존 정렬·접이식 동작이 회귀 없이 유지된다.

## Assumptions

- 워드마크는 우선 대시보드 홈 헤더에 적용한다. 모든 화면 공통 헤더로의 확대 적용은 본 범위 밖(후속 작업)으로 둔다.
- 워드마크 텍스트는 앱 이름 "ENUF"를 사용한다(layout의 metadata.title과 일치).
- 행 화살표가 여는 상세 시트는 005에서 만든 기존 상세 바텀시트(@sheet 슬롯)를 재사용한다. 신규 상세 화면을 만들지 않는다.
- 색상은 ENUF의 기존 primary(토스 블루 #3182f6)와 상승/하락(빨강/파랑) 토큰을 유지한다. 키움의 보라색 브랜드 컬러로 바꾸지 않는다(참고는 UI 구조와 어포던스이지 색 체계가 아니다).
- 다크 모드는 기존과 동일하게 범위 밖이다.
