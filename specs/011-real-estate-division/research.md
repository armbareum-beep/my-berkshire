# Phase 0 Research: 부동산 사업부

대화로 합의된 결정 기록. NEEDS CLARIFICATION 없음.

## Decision 1: 임대수익은 자체 원장(신규 테이블), events 미연동

**Decision**: 임대수익을 `manual_asset_income` 신규 테이블에 자산별로 기록. 주식 `events`에 넣지 않는다.

**Rationale**: 임대는 소득이지 자본투입(증자)이 아니다. events에 DEPOSIT으로 넣으면 투입원금·복리무중단이 깨지고, DIVIDEND는 종목 기반이라 의미 오염. 부동산은 이미 events와 분리된 수기 서브시스템이므로 대칭으로 자체 원장(헌장 V). 받은 임대를 주식에 재투자하면 그때 사용자가 별도 증자로 기록 → 이중계상 없음.

**Alternatives**: events 재사용(DEPOSIT/DIVIDEND) — 기각(위 이유). 자산 row에 누적 임대 단일 숫자만 — 기각(날짜별 이력·검증 불가).

## Decision 2: 세금·비용은 거래당 단일 합산 필드

**Decision**: 취득·임대·매도 각 거래에 비용을 **항목 분리 없이 한 필드**로. 주식 `feeAndTax`(InvestmentEvent) 단일 필드 패턴 그대로.

**Rationale**: 취득세·양도세·중개료·재산세를 따로 묻으면 입력 부담이 커 사용자가 안 쓴다. 사용자 명시 요구("너무 나누면 입력 힘드니까 한번에"). 모르면 0(가짜 추정 금지, 헌장 II).

**Alternatives**: 항목별 필드 — 기각(입력 마찰). 비용 무시 — 기각(수익률 왜곡).

## Decision 3: 비용의 회계 처리 — 취득은 원가 가산, 임대·매도는 실현 차감

**Decision**: 취득 부대비용 → **실질취득가(= 취득가 + 취득비용)** 로 분모에 가산. 임대비용·매도비용 → 해당 실현에서 차감. 모든 수익률 net.

**Rationale**: 취득세 등은 자본적 지출(원가의 일부)이므로 분모에. 양도세·중개료는 실현 수익을 줄이므로 분자에서 차감. 회계적으로 정확하고 주식 마찰비용 개념과 일관.

**Alternatives**: 모든 비용을 분자 차감(분모=취득가만) — 더 단순하나 취득세를 비용으로만 처리해 원가가 과소. 절충상 위가 정확.

## Decision 4: 매도 = 상태 전이(전체 매도), 보유에서 제외·실현 누적

**Decision**: 매도는 자산에 매도가·매도일·매도비용을 채워 표시. 매도일이 있으면 "매도됨" → 미실현(평가차익)에서 제외, **매도차익(= 매도가 − 실질취득가 − 매도비용) 실현**. 매도 이력 보존. 부분 매도는 범위 밖(전체 매도).

**Rationale**: 주식의 실현/미실현 대칭. 별도 boolean 없이 `sale_at` 유무로 상태 판정(단순). 부분 매도는 드물고 복잡 → 후순위.

**Alternatives**: 별도 sold 테이블 — 과설계. boolean 컬럼 — sale_at 유무로 충분.

## Decision 5: 평가 전부 수기 + 출처·평가일

**Decision**: 시세 자동연동 없음. 평가액은 수기, **평가 출처(KB/실거래가/감정가 등)·평가일** 입력·표시.

**Rationale**: KB시세 공개 API 없음. 국토부 실거래가 API는 아파트 한정(상가·토지 제외)이라 부분적·복잡도 대비 가치 낮음. 사용자도 수기 동의. 정직성은 출처·갱신일로 확보(vanity 방지).

**Alternatives**: 실거래가 API(아파트) — 기각(부분 커버, 후순위로만).

## Decision 6: 입력 안전장치(현재가 < 매입가 확인)

**Decision**: 현재가가 매입가보다 낮게 입력되면 저장 전 확인 안내.

**Rationale**: 사용자가 실제로 매입가/현재가를 바꿔 입력해 조용히 마이너스가 났다. 평가손실은 가능하나 흔치 않으니 한 번 확인이 실수를 잡는다.

## Decision 7: 홈 재배치 + XIRR 스코프 유지

**Decision**: 홈 `HoldingsStreamed` 순서를 보유계좌 → 주식 자산구성 → 부동산 사업부 → 현금 으로. **XIRR은 부동산 제외 유지**(시세 시계열 없음); 부동산은 누적(실현/미실현)만, 총자산 누적수익률엔 합산.

**Rationale**: 사용자 지정 순서, 사업부별 자산 흐름. XIRR은 머니가중 시계열이 필요한데 부동산은 일별 평가가 없어 부적합 — [[xirr-asset-scope]] 유지.

## Next.js 변형 주의
