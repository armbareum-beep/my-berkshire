# Quickstart: 부동산 사업부 검증

## 단위테스트 (Vitest — realAssets.test.ts / businessReturns.test.ts)

순수 계산 함수 검증:

1. **평가차익(미실현)**: 취득가 12.5억·현재가 18억(보유) → unrealized +5.5억, ret +44%.
2. **취득 부대비용**: 취득가 12.5억 + 취득비용 0.5억 → effectiveCost 13억, 현재가 18억 → unrealized +5억(+38.5%).
3. **임대 실현(net)**: 월 임대 500만 12건, 각 비용 50만 → realized += Σ(500−50)만 = 5,400만.
4. **매도차익(net)**: 취득가 12.5억(부대 0.5억)·매도가 18억·매도비용 0.5억 → saleGain = 18 − 13 − 0.5 = 4.5억, 매도 후 unrealized에서 제외.
5. **종합**: 매도차익 + 임대 + (보유분 평가차익) ÷ Σ실질취득가.
6. **events 불변**: 임대·매도 기록은 주식 XIRR·투입원금·복리무중단 입력에 안 들어감(별도 원장).
7. **취득가 없음**: effectiveCost null → 수익률 제외, 가치는 합산.
8. **businessReturns 통합**: manualGain=division.gain, manualCost=division.cost로 총자산 누적수익률에 반영.

품질 게이트: `npx tsc --noEmit` · `npx eslint` 클린.

## 수동 검증 (run/verify)

1. **입력 안전장치**: 매입가 18억·현재가 12.5억 입력 → 저장 전 "평가손실 맞나요?" 확인. (SC-005)
2. **평가차익**: 매입가 12.5억·현재가 18억 → 부동산 사업부 +5.5억(+44%). (SC-001)
3. **출처·평가일**: 출처(KB시세)·평가일 입력 → 카드/상세에 "추정 · KB시세 · {날짜}". (SC-004)
4. **임대수익**: 한 부동산에 임대 기록 → 실현(임대)에 누적, 미실현과 분리 표시. 주식 수익률/XIRR/복리무중단 **불변** 확인. (SC-002)
5. **매도**: 보유 부동산 매도 입력 → 미실현에서 사라지고 매도차익이 실현에, 매도 이력 보존. (SC-006)
6. **비용 net**: 비용 입력 시 수익률이 비용 차감 후로 줄고, 입력은 거래당 한 칸. (SC-007)
7. **홈 순서**: 보유계좌 → 주식 자산구성 → 부동산 사업부 → 현금. (SC-003)
8. **빈 사업부**: 부동산 없으면 홈에 부동산 사업부 카드 미표시.

## 회귀
- 주식 수익률·XIRR·복리무중단·총자산 누적수익률(부동산 없는 경우) 불변.
- 기존 수기자산 표시·순자산 합산 불변(컬럼 추가는 nullable, 하위호환).
