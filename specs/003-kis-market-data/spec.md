# Feature Specification: 한국투자증권(KIS) 시세·검색 데이터 소스 연동

**Feature Branch**: `003-kis-market-data`  
**Created**: 2026-06-22  
**Status**: Draft  
**Input**: User description: "한국투자증권(KIS) Open API를 시세·검색·환율 데이터 소스로 연동. FINANCE_SOURCE 플래그로 yahoo|kis 소스 교체 가능하게 하고 KIS를 1차 소스로 붙인다. 한글검색·국내/미국 시세·환율 동작. 야후는 펀더멘털·배당·미국ETF/지수 보조용으로 잔존. UI·DB·인터페이스 불변."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 한글로 종목을 검색한다 (Priority: P1)

사용자가 검색창에 "삼성전자", "에코프로" 같은 **한글 종목명**을 입력하면 해당 종목이 결과에 나온다. 현재(야후)는 한글 질의에 빈 결과를 줘서, 코드(005930)나 영문명을 알아야만 검색이 됐다.

**Why this priority**: 한국 사용자가 가장 자주 하는 행동이자 현재 가장 큰 결손. 이것만 고쳐도 즉시 체감 가치가 크다.

**Independent Test**: 검색창에 "삼성전자" 입력 → 삼성전자(005930)가 결과 상단에 표시되는지 확인. 다른 데이터(시세·펀더멘털) 없이도 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 검색 모달이 열린 상태, **When** "삼성전자"를 입력, **Then** 삼성전자(005930)가 결과에 포함된다.
2. **Given** 검색창, **When** "에코프로"를 입력, **Then** 에코프로 관련 종목이 결과에 나온다.
3. **Given** 검색창, **When** 종목코드 "005930" 또는 영문 "Samsung"을 입력, **Then** 기존처럼 정상 결과가 나온다(회귀 없음).

---

### User Story 2 - 공식 시세·환율로 평가액을 본다 (Priority: P2)

사용자의 보유 종목 현재가, 미국 종목 가격, 원/달러 환율이 **공식 소스(KIS)** 값으로 표시된다. 대시보드 평가액·수익률이 그 값으로 계산된다.

**Why this priority**: 검색이 진입점이라면 시세는 핵심 데이터. 비공식 야후 대신 증권사 공식값으로 신뢰도가 올라간다.

**Independent Test**: 보유 종목 1개(국내+미국)에 대해 표시 현재가가 KIS 조회값과 일치하는지, 환율이 일치하는지 확인.

**Acceptance Scenarios**:

1. **Given** 국내 종목(005930)·미국 종목(AAPL) 보유, **When** 대시보드를 연다, **Then** 각 현재가가 KIS 값과 일치한다.
2. **Given** 외화(USD) 보유 종목, **When** ₩ 환산 평가액을 본다, **Then** KIS 환율로 환산된 값이 표시된다.
3. **Given** 종목 상세의 과거 차트, **When** 연말 종가 추이를 본다, **Then** KIS 과거시세로 정상 렌더된다.

---

### User Story 3 - 소스를 안전하게 전환·되돌린다 (Priority: P3)

운영자가 설정값 하나로 데이터 소스를 `yahoo`↔`kis`로 전환할 수 있고, 전환해도 화면·저장 데이터·동작은 동일하다. KIS가 일시 장애일 때 기존 소스로 즉시 되돌릴 수 있다.

**Why this priority**: 비공식→공식 전환의 리스크를 줄이는 안전장치. 사용자 직접 가치는 아니지만 도입 신뢰성을 보장.

**Independent Test**: 설정을 `yahoo`로 두면 기존과 100% 동일, `kis`로 바꾸면 KIS 값으로 동작하는지 토글로 확인.

**Acceptance Scenarios**:

1. **Given** 소스 설정이 `yahoo`, **When** 앱을 사용, **Then** 기존 동작과 완전히 동일하다(회귀 없음).
2. **Given** 소스 설정이 `kis`, **When** 검색·시세를 사용, **Then** KIS 값으로 동작한다.
3. **Given** `kis` 설정 중 KIS 조회 실패, **When** 시세를 요청, **Then** 화면이 깨지지 않고 빈/폴백 상태로 안전하게 처리된다.

### Edge Cases

- KIS 인증 토큰 만료/실패 시: 시세·검색이 throw 하지 않고 빈 결과 또는 기존 소스 폴백으로 처리.
- KIS가 다루지 않는 심볼(지수 `^KS11`, 환율 `USDKRW=X`, 코인 `BTC-USD`): 시세전용 심볼은 기존 소스(야후) 경로 유지.
- KIS 레이트리밋 초과 시: 캐시(기존 revalidate)로 호출 빈도 억제, 초과분은 폴백.
- 토스 승인 시: 동일 플래그로 `kis`→`toss` 추가 전환이 가능해야 함(확장성).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 한글 종목명 검색에 대해 일치하는 한국 종목을 반환해야 한다.
- **FR-002**: 시스템은 종목코드·영문명 검색에 대해 기존과 동일한 결과를 유지해야 한다(회귀 금지).
- **FR-003**: 시스템은 국내·미국 종목의 현재가를 공식 소스(KIS)에서 가져와 표시·계산에 사용해야 한다.
- **FR-004**: 시스템은 원/달러 등 환율을 공식 소스에서 가져와 ₩ 환산에 사용해야 한다.
- **FR-005**: 시스템은 과거 종가 시계열을 공식 소스에서 가져와 차트·추이 계산에 사용해야 한다.
- **FR-006**: 시스템은 단일 설정값으로 데이터 소스를 전환할 수 있어야 하며, 기본값은 기존 소스(yahoo)여야 한다.
- **FR-007**: 데이터 소스를 전환해도 UI·저장 데이터·인터페이스 계약은 변하지 않아야 한다.
- **FR-008**: 시스템은 펀더멘털·배당이력·미국 ETF/지수 구성 데이터는 기존 소스(야후/DART/EDGAR/KRX)를 그대로 사용해야 한다(이번 범위 밖).
- **FR-009**: 공식 소스 조회 실패 시 화면이 깨지지 않고 안전하게(빈/폴백) 처리되어야 한다.
- **FR-010**: 인증 자격증명(KIS 앱키/시크릿)은 서버에서만 사용되며 클라이언트에 노출되지 않아야 한다.

### Key Entities *(include if feature involves data)*

- **종목 검색 결과**: 종목코드(내부심볼), 이름, 거래소, 자산유형. 외부 소스가 바뀌어도 동일한 형태를 유지.
- **시세**: 종목코드별 현재가·전일종가·통화. ₩ 환산값과 native 값 구분.
- **환율**: 통화→KRW 비율.
- **데이터 소스 설정**: 현재 활성 소스(yahoo|kis|…)를 가리키는 단일 값.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 한글 종목명 검색 성공률 0% → 주요 한국 상장 종목에 대해 95%+ 결과 반환.
- **SC-002**: 표시 현재가가 공식 소스 값과 동일(허용 오차: 시점차 외 0).
- **SC-003**: 소스 설정 `yahoo` 상태에서 기존 동작 대비 회귀 0건.
- **SC-004**: 소스 장애 시 사용자에게 보이는 오류·빈화면 크래시 0건(안전 폴백).
- **SC-005**: 펀더멘털·배당·ETF/지수 화면은 변경 전과 동일하게 동작(회귀 0건).

## Assumptions

- KIS 자격증명은 본인 계좌 기준으로 발급되어 있고, 시세·검색 조회에 충분한 권한·레이트리밋을 가진다(토큰 발급 검증 완료).
- 이 단계의 사용 범위는 본인/비공개 베타이며, 멀티유저 공개 시 시세 재배포 라이선스(코스콤)는 별도 사업 트랙으로 처리한다(이번 범위 밖).
- 지수·환율·코인 등 "시세전용" 심볼은 KIS 범위 밖일 수 있어 기존 소스 경로를 유지한다.
- 상세 기술 설계·소스 매핑은 기존 문서(`docs/toss-migration-spec-v1.md`, `docs/api-design-spec-v1.md`)를 근거로 `/speckit.plan` 단계에서 구체화한다.
- 검색 데이터가 텍스트 쿼리 검색인지 종목마스터 다운로드인지는 plan 단계에서 KIS 스펙 확인 후 결정한다(마스터면 로컬 인덱싱).
