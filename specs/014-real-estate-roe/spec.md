# Feature Specification: 부동산 사업부 실투자금 수익률·순자산·LTV

**Feature Branch**: `014-real-estate-roe`  
**Created**: 2026-06-26  
**Status**: Draft  
**Input**: User description: "부동산 사업부 실투자금 수익률·순자산·LTV — 대출(레버리지) 데이터로 실투자금 기준 손익 표시"

## 용어 정의

- **실투자금**: 내가 실제로 넣은 현금 = (취득가 + 취득 부대비용 + 자본투입) − 대출잔액. 대출로 충당한 금액은 제외한 "내 돈".
- **실투자금 수익률**: 누적손익 ÷ 실투자금. 분자(누적손익) = 평가차익 + 임대순익 + 매도차익 − 이자.
  - 임대료가 없어도(거주용·시세차익 목적) 분자는 그냥 **평가차익 − 이자**가 된다. "임대 수익"을 전제하지 않는다 — 대출을 끼고 산 자산의 가치가 내 돈 대비 얼마나 불었는지를 본다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 레버리지 반영 실투자금 수익률 보기 (Priority: P1)

부동산을 대출을 끼고 산 사용자는, 부동산 사업부 상세 화면에서 "내가 실제 넣은 돈(실투자금) 대비 얼마나 불었나"를 한눈에 본다. 기존 자산수익률(취득가 기준)은 레버리지를 무시하므로, 같은 자리에 실투자금 수익률을 나란히 보여 레버리지 효과를 드러낸다. 임대료가 없는 거주용·시세차익 목적이어도 분자가 평가차익으로 잡혀 그대로 의미가 있다.

**Why this priority**: 사용자가 직접 요청한 핵심 가치. 대출 데이터(spec 012)가 이미 있으므로, 분모를 실투자금으로 바꾸는 것만으로 "진짜 내 돈 대비 수익률"을 보여줄 수 있다. 이 하나만으로도 독립적인 가치를 전달한다.

**Independent Test**: 대출이 연결된 부동산 물건 1건을 등록한 뒤 상세 화면에서 자산수익률과 실투자금 수익률이 나란히 표시되고, 실투자금 수익률이 자산수익률보다 레버리지 배율만큼 증폭되어 나타나는지 확인.

**Acceptance Scenarios**:

1. **Given** 10억 부동산을 자기돈 3억 + 대출 7억으로 취득해 평가액이 11억으로 올랐고 이자가 일부 발생한 상태, **When** 부동산 사업부 상세 화면을 연다, **Then** 자산수익률(취득가 기준)과 실투자금 수익률(실투자금 기준)이 나란히 표시되고 실투자금 수익률이 자산수익률보다 높게(증폭되어) 나타난다.
2. **Given** 임대료 없는 거주용 부동산을 대출 끼고 보유(평가차익만 발생), **When** 상세 화면을 연다, **Then** 분자는 평가차익 − 이자로 잡혀 실투자금 수익률이 정상 표시된다(임대수익 0이어도 "—"가 아니다).
3. **Given** 대출이 전혀 없는 부동산만 보유, **When** 상세 화면을 연다, **Then** 실투자금 수익률/LTV 지표 묶음은 표시되지 않는다(자산수익률은 기존대로 표시).
4. **Given** 부동산 외 사업부(대체·사업)만 보유, **When** 해당 사업부를 본다, **Then** 실투자금 수익률/순자산/LTV는 표시되지 않는다.

---

### User Story 2 - 내 순자산과 레버리지 비율(LTV) 보기 (Priority: P2)

사용자는 부동산 사업부에서 "이 부동산에서 지금 내 몫이 얼마인지(순자산 = 평가액 − 대출잔액)"와 "얼마나 빚을 끼고 있는지(LTV = 대출 / 평가액)"를 함께 본다.

**Why this priority**: 실투자금 수익률의 맥락을 보강하는 보조 지표. 수익률 수치 하나만으로는 위험(레버리지 과다)을 가늠하기 어려우므로 순자산·LTV가 균형을 준다. 같은 데이터에서 바로 파생되어 추가 비용이 거의 없다.

**Independent Test**: 대출 연결 부동산에서 순자산이 (평가액 − 대출잔액)과 일치하고, LTV가 (대출잔액 / 평가액)로 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 평가액 11억·대출잔액 7억인 부동산, **When** 상세 화면을 연다, **Then** 순자산 4억, LTV 약 64%로 표시된다.
2. **Given** 대출잔액이 평가액 이상(과레버리지)인 상태, **When** 상세 화면을 연다, **Then** 순자산은 음수로 표시되고 실투자금 수익률은 산출 불가("—")로 표시된다.

---

### User Story 3 - 물건별 지표를 탭으로 펼쳐 보기 (Priority: P3)

사용자는 사업부 합산 strip은 그대로 두되, 개별 물건을 **탭하면** 그 물건만의 실투자금 수익률·순자산·LTV와 연결 대출 내역이 아래로 펼쳐지길 원한다. 평소 화면은 단순하게(점진적 공개), 궁금한 물건만 깊이 본다.

**Why this priority**: US1·US2(사업부 합산)가 헤드라인이고, 물건별은 심화. 모두 펼치면 정보 과다 → 헌장 III(화면 단순·점진적 공개)에 따라 탭 시 노출. 1~2채 가정에서 합산이 기본, 물건별은 on-demand.

**Independent Test**: 부동산 2채(한 채만 대출) → 물건 행 탭 시 그 물건의 실투자금 수익률·LTV·순자산·연결 대출이 펼쳐지고, 다시 탭하면 접힘. 합산 strip은 불변.

**Acceptance Scenarios**:

1. **Given** 대출 연결된 부동산 물건, **When** 그 행을 탭한다, **Then** 그 물건의 자산수익률·실투자금 수익률·순자산·LTV와 연결 대출 목록(이자·수정·삭제)이 아래로 펼쳐진다.
2. **Given** 펼쳐진 물건, **When** 다시 탭한다, **Then** 접혀서 기본(이름·평가액·미실현)만 남는다.
3. **Given** 대출 없는 물건, **When** 탭한다, **Then** 물건별 레버리지 지표(실투자금 수익률·LTV)는 표시되지 않는다(자산수익률·평가 출처만).
4. **Given** 미연결 공통 대출이 있는 사업부, **When** 물건별 합을 본다, **Then** 물건별 합 ≠ 사업부 합일 수 있으며, 미연결 공통 대출은 사업부 합산에만 반영됨이 안내된다.

---

### Edge Cases

- **실투자금 ≤ 0**(대출이 취득원가 이상): 분모가 0 이하 → 실투자금 수익률은 "—"(산출 불가)로 표시한다.
- **평가액 0**(보유 물건 없음 또는 전부 매도): LTV 산출 불가 → LTV는 "—"로 표시.
- **임대료 0**(거주용·시세차익 목적): 분자는 평가차익 − 이자로 계산 — 수익률은 정상 표시된다(0이나 "—"가 아니다).
- **취득가 미입력 자산**: 기존 정책대로 수익률 스코프 밖이나, 평가액은 순자산(평가액 − 대출)에는 합산된다.
- **대출 0**: 실투자금 = 취득원가 → 실투자금 수익률 = 자산수익률, LTV 없음 → 지표 묶음 자체를 숨긴다(중복·잡음 방지).
- **여러 대출**(연결 + 사업부 공통): 사업부 대출잔액 합으로 실투자금·LTV를 계산한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 부동산 사업부의 **실투자금 수익률** = (이자 차감 후 누적손익) ÷ (실투자금)을 계산해야 한다. 실투자금 = (취득가 + 취득 부대비용 + 자본투입) − 대출잔액. 누적손익 = 평가차익 + 임대순익 + 매도차익 − 이자.
- **FR-002**: 시스템은 부동산 사업부의 **순자산** = (보유 물건 현재 평가액 합) − (대출잔액 합)을 계산해야 한다.
- **FR-003**: 시스템은 부동산 사업부의 **LTV(레버리지 비율)** = 대출잔액 합 ÷ 보유 물건 현재 평가액 합을 계산해야 한다.
- **FR-004**: 누적손익은 이미 대출 이자가 차감된 값(spec 012)을 사용해야 한다 — 분자에 이자가 이중 반영되거나 누락되지 않아야 한다.
- **FR-005**: 사용자는 부동산 사업부 **상세 화면**(`/real-estate`)에서 기존 자산수익률과 실투자금 수익률을 **나란히**, 그리고 순자산·LTV를 함께 볼 수 있어야 한다.
- **FR-006**: 시스템은 대출이 있는 부동산 사업부에서만 실투자금 수익률/순자산/LTV 묶음을 노출해야 한다. 대출이 없거나 부동산 외 사업부에서는 노출하지 않는다.
- **FR-007**: 홈 통합 "실물 사업부" 카드는 변경하지 않고 기존 자산수익률 한 줄만 유지해야 한다(사업부 간 시각적 일관성).
- **FR-008**: 실투자금이 0 이하이면 실투자금 수익률을 산출하지 않고 "—"로 표시해야 한다. 평가액이 0이면 LTV를 "—"로 표시해야 한다.
- **FR-009**: 임대수익이 없어도(거주용·시세차익) 실투자금 수익률은 평가차익 − 이자로 산출되어 표시되어야 한다 — "수익률"이 임대 수익을 전제하지 않는다.
- **FR-010**: 이 기능은 표시 전용이어야 한다 — 주식 XIRR, 총자산 누적수익률 합산 로직, 저장 데이터(스키마)를 변경하지 않는다.
- **FR-011**: 시스템은 **물건별** 실투자금 수익률·순자산·LTV를 그 물건의 취득원가·평가액·**연결 대출**(`manualAssetId` 연결분)로 계산해야 한다. 미연결 공통 대출은 물건별에 포함하지 않고 사업부 합산에만 반영한다.
- **FR-012**: 물건 행은 기본 접힘(이름·평가액·미실현)이고, **탭하면** 물건별 지표·연결 대출 내역이 펼쳐져야 한다(점진적 공개). 사업부 합산 strip은 항상 표시(불변).
- **FR-013**: 대출 없는 물건은 펼쳐도 물건별 레버리지 지표(실투자금 수익률·LTV)를 표시하지 않는다.

### Key Entities *(include if feature involves data)*

- **부동산 사업부 집계**: 한 사업부의 취득원가 합·평가차익·실현손익·대출잔액·평가액 합에서 파생되는 표시 지표(자산수익률, 실투자금 수익률, 순자산, LTV). 저장하지 않고 조회 시 계산.
- **담보대출(기존)**: 물건에 연결되거나 사업부 공통으로 묶인 대출. 잔액·이율 보유. 실투자금(분모)과 LTV·순자산 계산의 입력.
- **부동산 물건(기존)**: 취득가·부대비용·현재 평가액·매도 여부 보유. 실투자금·평가액·순자산 계산의 입력.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 대출 연결 부동산 물건이 1건 이상 있을 때, 사용자는 상세 화면에서 실투자금 수익률·순자산·LTV를 추가 조작 없이 즉시 확인할 수 있다.
- **SC-002**: 레버리지가 있는 경우 실투자금 수익률은 자산수익률과 다른 값(증폭)을 보여, 레버리지 효과가 수치로 구분된다.
- **SC-003**: 순자산 표시값이 (평가액 − 대출잔액)과, LTV가 (대출잔액 ÷ 평가액)과 사용자가 손으로 계산한 값과 일치한다(표시 반올림 이내).
- **SC-004**: 임대료가 없는 거주용 부동산에서도 실투자금 수익률이 "—"가 아닌 실제 값(평가차익 기반)으로 표시된다.
- **SC-005**: 대출이 없는 사업부·물건에서는 실투자금 수익률/LTV 묶음이 나타나지 않아, 화면이 기존과 동일하게 단순하게 유지된다(불필요한 지표 0건).
- **SC-006**: 기존 자산수익률·총자산 누적수익률·주식 XIRR 표시값이 본 기능 도입 전후로 동일하다(회귀 0건).

## Assumptions

- "실투자금 수익률"의 **수익률**은 임대 수익이 아니라 시세차익을 포함한 총손익률을 뜻한다 — 거주용·차익 목적 사용자도 대상으로 본다(사용자 피드백 반영).
- 실투자금 분모의 대출잔액은 **현재 잔액**을 원차입액 근사로 사용한다(개인 가계부 수준 정확도 허용; 모기지 상환은 느려 근사 오차 작음).
- 자본투입(capital, spec 012 보정)은 사용자가 추가로 넣은 자기 돈으로 보아 실투자금에 가산한다.
- 실투자금 수익률/순자산/LTV는 **부동산(REAL_ESTATE) 사업부 전용**이다. 현재 레버리지(담보대출)는 부동산에만 연결되므로 다른 사업부에는 적용 대상이 없다.
- 표시 통화 환산(₩/$ 토글)·반올림은 기존 포맷 규칙을 그대로 따른다.
- 기존 대출 연결 데이터(spec 012: 물건↔대출 연결, 이자 파생, 보정 체크포인트)를 재사용하며 신규 입력 화면은 추가하지 않는다.
