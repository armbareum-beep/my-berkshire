# Feature Specification: 거래내역 정밀도 복원 (연혁 복원 게이미피케이션)

**Feature Branch**: `001-import-fidelity`
**Created**: 2026-06-22
**Status**: Draft
**Input**: 거래내역 백필을 "회사 연혁 복원" 게임으로 만들되, 온보딩 현재보유 스냅샷을 베이스라인으로 두고 종목별 실제 매매로 교체·정합(순수량=보유), 매도완료 종목은 선택 입력(실현손익 잠금해제), 회사 나이·정밀도 미터·첫 거래 선언으로 설립 확정.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 종목별 기록 복원과 정합 (Priority: P1)

사용자는 온보딩에서 현재 보유 종목을 평단으로 간단히 입력했다(스냅샷). 이제 한 종목을 골라 실제 매수·매도 내역을 그 자리에서(화면 이동 없이) 입력한다. 입력하는 동안 "입력 N주 ↔ 보유 M주"가 실시간으로 보이고, 두 수가 일치하면 그 종목의 임시 스냅샷 기록이 실제 내역으로 **교체**된다. 보유 수량·자산 평가액은 변하지 않는다(이중 계상 없음).

**Why this priority**: 데이터 정합이 모든 수치(자산·수익률)의 신뢰 기반이다. 스냅샷 위에 역사를 그냥 더하면 포지션이 두 배로 잡혀 제품이 깨진다. 이 스토리만 구현해도 "정확한 종목별 원장"이라는 독립적 가치가 선다.

**Independent Test**: 보유 종목 하나에 대해 실제 매매를 입력해 순수량을 보유와 맞춘 뒤 교체를 실행 → 보유 수량 불변·스냅샷 사라짐·실제 내역만 남음을 확인.

**Acceptance Scenarios**:

1. **Given** 스냅샷으로 50주 보유 중인 종목, **When** 실제 매수들을 합쳐 순 50주를 입력하고 교체를 실행, **Then** 그 종목의 보유는 여전히 50주이고 임시 스냅샷 기록은 사라지고 실제 내역으로 대체된다.
2. **Given** 같은 종목, **When** 실제 입력 순수량이 30주(보유 50주와 불일치), **Then** 교체가 막히고 "입력 30 ↔ 보유 50" 차이를 중립적으로 안내하며 어떤 기록도 삭제되지 않는다.
3. **Given** 어느 연도 입력 화면, **When** 사용자가 그 화면을 열면, **Then** 연도와 무관하게 자신이 이미 입력한 매수·매도 전체가 보여 같은 거래를 중복 입력하지 않는다.

---

### User Story 2 - 연혁 복원 동기(회사 나이·정밀도·설립 확정) (Priority: P2)

사용자가 과거 거래를 넣을수록 회사가 "더 오래된 회사"가 된다(트랙레코드 기간↑). 정밀도 미터(복원된 종목 비율)가 차오르고, 충분히 복원되면 실제 수익률 지표가 잠금 해제된다. 사용자는 "이게 내 첫 거래"라고 선언해 연혁 복원을 완료(설립 확정)할 수 있다.

**Why this priority**: 정합(P1)이 정확성을 보장한다면, 이 동기 레이어가 지루한 수기 입력을 실제로 하게 만든다. 단, 정직해야 한다(입력량이지 성과가 아님).

**Independent Test**: 종목 N개 중 2개를 복원 → 정밀도 미터가 2/N로 갱신되고, 회사 나이가 가장 이른 실제 거래일 기준으로 표시되며, 설립 확정 시 완료 상태가 됨을 확인.

**Acceptance Scenarios**:

1. **Given** 5개 보유 종목 중 2개 복원, **When** 정밀도를 보면, **Then** "정밀도 2/5"로 정확히 표시되고 미복원 종목은 중립 톤("스냅샷")으로 표기된다(죄책감 유발 문구 없음).
2. **Given** 2019년 실제 매수를 입력, **When** 회사 나이를 보면, **Then** 설립일이 2019년으로 당겨지고 트랙레코드 기간이 그에 맞게 늘어난다.
3. **Given** 설립 확정 후, **When** 사용자가 그보다 더 이른 거래를 입력, **Then** 시스템은 확정을 자동 해제하고 설립일을 갱신하며 그 사실을 알린다.
4. **Given** 실제 수익률 지표가 아직 잠긴 상태, **When** 화면을 보면, **Then** 잠금 자리에는 가짜 수치가 아니라 가림 표시와 "무엇을 더 채우면 열리는지" 안내가 보인다.

---

### User Story 3 - 이미 판 종목(왕복) 선택 입력 → 실현손익 (Priority: P3)

사용자는 이미 매도해 더 이상 보유하지 않는 종목의 매수·매도(왕복)를 **선택적으로** 입력할 수 있다. 필수가 아니며, 입력하면 실현손익(이미 실현한 수익)이 잠금 해제되어 보인다.

**Why this priority**: 완전 복원의 정확성은 높지만 진입장벽이 크다. 그래서 강제하지 않고, 넣은 사람에게만 추가 보상(실현손익·더 정확한 장기 수익률)을 주는 상위 정밀도 티어로 둔다.

**Independent Test**: 미보유 종목의 매수→매도를 동수로 입력(순 0) → 실현손익 카드가 잠금 해제되고 표시 금액이 평균원가 기준과 일치함을 확인.

**Acceptance Scenarios**:

1. **Given** 보유하지 않는 종목, **When** 매수·매도를 순 0이 되게 입력, **Then** 그 왕복이 기록되고 실현손익 지표가 잠금 해제된다.
2. **Given** 사용자가 매도완료 종목을 전혀 넣지 않음, **When** 앱을 사용, **Then** 보유 종목 손익은 정상 동작하고 어디서도 입력을 강요하지 않는다.

---

### Edge Cases

- **수량 불일치(과/소 입력)**: 교체를 막고 차이를 안내, 자동 삭제 없음(US1-2).
- **매도완료 순수량 ≠ 0**: 일부만 매도해 잔여가 있으면 그것은 보유 종목(P1) 처리이며 왕복(P3) 대상이 아니다.
- **설립 확정 후 더 이른 거래 추가**: 확정 자동 해제 + 설립일 갱신 + 안내(US2-3).
- **레거시 회사(스냅샷 표시가 없는 기존 데이터)**: 교체 흐름 대신 일반 입력으로 동작하고 중립 카피로 안내(크래시·오작동 없음).
- **복원 후 실제 거래를 삭제**: 표시 보유가 남은 실제 내역의 순수량을 그대로 따라가 항상 자기일관.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 온보딩에서 입력한 현재 보유를 "임시(스냅샷) 기록"으로 보존하고, 정밀 복원 전까지 보유·자산이 정상 표시되어야 한다.
- **FR-002**: 사용자는 한 종목의 실제 매수/매도를 화면 이동 없이 입력할 수 있고(저장은 "복원 완료" 전까지 화면에만 보관), 시스템은 입력 순수량과 스냅샷이 주장한 보유 수량을 "입력 N ↔ 보유 M"로 실시간 비교해 보여야 한다.
- **FR-003**: 입력 순수량이 스냅샷 보유 수량과 일치할 때에만, 모아온 실제 매매를 한 번에 저장하고 동시에 임시(스냅샷) 기록을 삭제한다(일시적 이중 계상 없음). 보유 수량은 보존되되, **평단가(원가)는 실제 기록으로 교체되어 정확해진다.**
- **FR-004**: 순수량 불일치 시 교체를 막고 차이를 중립적으로 안내해야 하며, 어떤 기록도 자동 삭제하지 않는다.
- **FR-005**: 시스템은 가장 이른 실제 거래일을 회사 설립일로 반영해야 하고(과거로만 이동), 회사 나이(트랙레코드 기간)를 그에 맞춰 표시한다.
- **FR-006**: 시스템은 정밀도(복원된 종목 비율)를 정직하게 보여야 한다 — 성과가 아니라 입력량이며, 미복원/빈 상태는 중립 톤으로 표기(죄책감 유발 금지).
- **FR-007**: 충분한 실제 기록이 쌓이면 실제 수익률 지표를 잠금 해제해 보여야 하고, 잠금 상태에서는 가짜 수치를 노출하지 않는다.
- **FR-008**: 사용자는 미보유(매도완료) 종목의 왕복 내역을 선택적으로 입력할 수 있어야 하며(필수 아님), 입력 시 실현손익이 잠금 해제된다.
- **FR-009**: 사용자는 "이게 내 첫 거래"임을 선언해 연혁 복원을 완료(설립 확정)할 수 있어야 하고, 시스템은 사용자가 선언하지 않은 설립 시점을 임의로 단정하지 않는다.
- **FR-010**: 설립 확정 후 더 이른 거래가 입력되면 시스템은 확정을 자동 해제하고 설립일을 갱신하며 사용자에게 알린다.
- **FR-011**: 직접 입력 전, 사용자가 자기 증권사에서 거래내역을 어디서 조회하는지 증권사별 안내를 제공한다.
- **FR-012**: 점수·미터·보상은 매매 스타일이나 거래 빈도를 평가하지 않는다(보편 규율·기록 완성도만).

### Key Entities *(include if feature involves data)*

- **거래 기록**: 매수·매도·배당·현금 흐름의 단일 항목. 종목·수량·가격·일자, 그리고 출처(임시 스냅샷 vs 실제 입력)를 구분.
- **보유 포지션**: 종목별 현재 순보유 수량(실제 기록의 매수−매도 합).
- **회사(지주회사)**: 설립일(가장 이른 기록), 설립 확정 여부, 연도별 입력 완료 표시.
- **종목 정밀도 상태(티어)**: 임시(스냅샷) / 복원완료(보유·정합) / 매도완료(왕복).

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 보유 종목 하나를 화면 이동 없이 3분 이내에 실제 내역으로 복원할 수 있다.
- **SC-002**: 복원 후 종목의 표시 보유 수량이 사용자가 입력한 실제 순수량과 100% 일치한다(이중 계상 0건).
- **SC-003**: 사용자는 어느 연도 화면에서든 자신의 기존 매수·매도 전체를 볼 수 있어 같은 거래를 중복 입력하는 사례가 없다.
- **SC-004**: 정밀도 미터와 회사 나이가 실제 입력에 따라 정확히 갱신된다(예: 5종목 중 2종목 복원 → 정밀도 40%).
- **SC-005**: 잠긴 지표 자리에 실제값이 아닌 임의 숫자가 노출되는 사례가 0건이다.
- **SC-006**: 매도완료 종목 입력 시 표시되는 실현손익이 평균원가 기준 계산과 일치한다.

## Assumptions

- 대상은 장부(ledger) 모드 회사다. 챌린지/라이브 모드의 소급 입력은 비범위.
- 기능통화는 KRW이며 외화는 환산해 표시한다.
- 직접 입력 UI, 증권사 조회 안내, 연도별 입력 구조는 기존 기능을 재사용·확장한다(이번 기능은 정합·정밀도·게이미피케이션 레이어를 더한다).
- 레거시 회사(임시 스냅샷 표시가 없는 기존 데이터)는 교체 흐름 없이 일반 입력으로 동작하며 중립적으로 안내한다.
- 시세·종목명은 기존 소스를 통해 제공된다.
- 파일(CSV/Excel) 업로드 경로는 비범위(직접 입력으로 일원화).
