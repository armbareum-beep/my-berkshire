# Feature Specification: 트랜잭션·자산배분 화면 종목 로고 적용

**Feature Branch**: `013-logo-missing-pages`  
**Created**: 2026-06-25  
**Status**: Draft  
**Input**: User description: "로고가 트랜젝션이랑 return allocation 페이지에서는 적용 안되고있어" + 추가: "account에 계좌에도 로고가 있어야해 해당증권사 로고"

## 배경 *(context)*

앱의 다른 화면(보유 종목 목록, 수익률/Returns 목록, 종목 검색)에서는 각 종목이 자기
브랜드 로고가 들어간 원형 아이콘으로 표시된다. 그러나 **거래 내역(Transactions)** 화면과
**자산배분(Allocation) 화면**에서는 같은 종목인데도 로고가 보이지 않고, 거래 유형 아이콘(매수/
매도/배당 등의 모노톤 기호)이나 글자 이니셜·색 점으로만 표시된다. 사용자는 한 종목을 화면마다
다른 모습으로 보게 되어 "어떤 종목인지" 시각적으로 즉시 알아보기 어렵다.

또한 계좌(Accounts) 화면의 각 계좌는 현재 증권사 이니셜+브랜드 컬러 배지로만 표시되는데,
사용자는 각 계좌에 **해당 증권사의 로고**가 보이기를 원한다.

이 기능은 종목을 나열하는 모든 화면에서 로고 표시를 **일관되게** 맞추고, 계좌에는 증권사 로고를
적용하는 것을 목표로 한다.

## Clarifications

### Session 2026-06-25

- Q: 거래 내역에서 종목 로고를 유형 아이콘과 어떻게 배치할까(대체/병기/별도)? → A: 새 배치·배지를
  발명하지 않고, **다른 화면(보유·수익률)이 쓰는 기존 로고 표시 방식을 그대로 재사용**한다. 즉
  종목 로고를 공용 아바타로 메인 표시하고, 거래 유형은 기존 텍스트 라벨로 구분(신규 코너 배지
  추가 안 함).
- Q: 증권사 로고는 어디서 가져올까(소싱 방식)? → A: 종목 로고와 **동일한 기존 파이프라인을 그대로
  따른다** — 셀프 호스팅 에셋 우선, 없으면 증권사 도메인 favicon 폴백(앱이 운용사/ETF 로고에 이미
  쓰는 방식과 동일), 그래도 없으면 기존 이니셜+컬러 배지. 새로운 외부 의존·새 표시 규칙을 만들지
  않는다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 자산배분 화면에서 종목 로고 보기 (Priority: P1)

사용자가 자산배분 화면(전체 종목 목록, 슬리브별, 태그별)을 열면 각 종목 행에 해당 종목의
브랜드 로고가 보유 목록에서 보던 것과 동일하게 표시된다.

**Why this priority**: 자산배분은 포트폴리오 구성을 한눈에 파악하는 핵심 화면이고, 현재 글자
이니셜만 나와 종목 식별성이 가장 떨어진다. 데이터(종목 코드)는 이미 행마다 존재하므로 가장
빠르게 가치를 전달한다.

**Independent Test**: 둘 이상의 종목을 보유한 포트폴리오로 자산배분 종목 목록을 열어, 각 행의
아이콘이 보유 목록 화면과 동일한 로고로 나오는지 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 로고가 표시되는 종목(예: 보유 목록에서 로고가 보이는 종목)을 보유한 상태에서,
   **When** 자산배분 종목 목록 화면을 열면, **Then** 그 종목 행의 아이콘이 보유 목록과 같은
   브랜드 로고로 표시된다.
2. **Given** 로고 파일이 없는 종목을 보유한 상태에서, **When** 자산배분 화면을 열면, **Then**
   보유 목록과 동일한 이니셜·색 폴백으로 표시된다(빈 칸·깨진 이미지 없음).
3. **Given** "현금" 등 종목이 아닌 행이 목록에 포함된 경우, **When** 화면을 열면, **Then** 현금
   행은 기존 표현을 유지하고 잘못된 로고를 끌어오지 않는다.

---

### User Story 2 - 거래 내역에서 종목 식별하기 (Priority: P2)

사용자가 거래 내역을 볼 때 각 거래가 어떤 종목/대상에 대한 것인지 로고로 즉시 알아본다.

**Why this priority**: 거래 내역은 종목 식별보다 "무슨 일이 있었는지(매수/매도/배당/입출금)"가
먼저인 화면이고, 입출금·환전처럼 종목이 없는 거래가 섞여 있어 처리 규칙이 더 필요해 P1보다 뒤에
둔다. 배치 방식은 명확화됨 — 다른 화면과 동일한 공용 로고 표시를 그대로 재사용한다(Clarifications).

**Independent Test**: 한 종목에 대한 매수·매도·배당 거래와 입출금·환전 거래가 섞인 내역을 열어,
종목 거래에는 종목이 식별되는 시각 요소가, 비종목 거래에는 기존 표현이 유지되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 특정 종목의 매수/매도/배당 거래가 있을 때, **When** 거래 내역을 열면, **Then** 그
   거래에서 어떤 종목인지 시각적으로(로고 포함) 식별할 수 있다.
2. **Given** 입금·출금·환전처럼 종목이 없는 거래가 있을 때, **When** 거래 내역을 열면, **Then**
   해당 거래는 종목 로고 없이 기존 유형 표현을 유지한다.
3. **Given** 거래 유형 정보(매수/매도/배당 등)도 사용자에게 여전히 중요할 때, **When** 종목
   로고를 적용해도, **Then** 거래 유형을 구분하는 정보가 사라지지 않는다.

---

### User Story 3 - 계좌에 증권사 로고 보기 (Priority: P2)

사용자가 계좌 화면을 열면 각 계좌가 연결된 증권사의 로고로 표시되어, 어느 증권사 계좌인지
한눈에 알아본다.

**Why this priority**: 계좌 식별성을 높이고 종목 화면과 같은 "로고가 있는" 시각 언어로 통일한다.
다만 증권사 로고 에셋은 현재 라이선스 부담을 이유로 의도적으로 이니셜+컬러로 대체해 온 영역이라,
에셋 확보 여부에 따라 폴백 동작이 중요해 P1보다 뒤에 둔다.

**Independent Test**: 서로 다른 두 증권사 계좌를 만들고 계좌 화면을 열어, 각 계좌가 해당 증권사
로고(또는 로고 부재 시 기존 이니셜+컬러)로 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 증권사가 지정된 계좌가 있을 때, **When** 계좌 화면을 열면, **Then** 그 계좌는 해당
   증권사의 로고로 표시된다.
2. **Given** 증권사 로고 에셋이 없는(또는 증권사 미지정) 계좌가 있을 때, **When** 계좌 화면을
   열면, **Then** 기존 이니셜+브랜드 컬러 배지로 안정적으로 표시된다(빈 칸·깨진 이미지 없음).

---

### Edge Cases

- 종목 코드는 있으나 로고 이미지가 존재하지 않는 종목 → 다른 화면과 동일한 이니셜·색 폴백.
- 종목이 없는 행/거래(현금, 입금, 출금, 환전) → 로고를 추측해 끌어오지 않음.
- 외화·해외·암호화폐·지수 등 종목 유형이 섞인 목록 → 각 유형의 기존 폴백 규칙을 그대로 따름.
- 로고 이미지 로드가 느리거나 실패 → 깨진 이미지가 아니라 폴백으로 안정적으로 대체.
- 증권사 미지정 계좌 또는 로고 에셋이 없는 증권사 → 기존 이니셜+브랜드 컬러 배지 유지.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 자산배분 화면의 종목 행에서, 앱의 다른 종목 목록 화면과 **동일한 규칙**으로
  종목 브랜드 로고를 표시해야 한다.
- **FR-002**: 시스템은 거래 내역 화면에서 종목과 연결된 거래에 대해, 다른 종목 목록 화면이 쓰는
  것과 **동일한 공용 로고 표시 방식**으로 종목 로고를 표시해야 한다(새 배치·배지를 발명하지 않음).
  거래 유형(매수/매도/배당/입출금/환전) 구분은 기존 텍스트 라벨로 계속 전달되어야 한다.
- **FR-003**: 종목이 없는 행·거래(현금, 입금, 출금, 환전)에 대해서는 로고를 표시하지 않고 기존
  표현을 유지해야 한다.
- **FR-004**: 로고 이미지가 없거나 로드에 실패하면, 시스템은 앱 전반에서 쓰는 동일한 이니셜·색
  폴백으로 대체해야 한다(빈 칸·깨진 이미지 금지).
- **FR-005**: 모든 종목 나열 화면(보유, 수익률, 자산배분, 거래 내역 등)은 같은 종목에 대해 동일한
  로고/폴백 결과를 보여야 한다(화면 간 불일치 금지).
- **FR-006**: 시스템은 계좌 화면에서 각 계좌를 연결된 증권사 로고로 표시해야 한다. 로고 소싱은
  종목 로고와 **동일한 기존 파이프라인**을 따른다 — 셀프 호스팅 에셋 우선, 없으면 증권사 도메인
  favicon 폴백, 그래도 없거나 증권사 미지정이면 기존 이니셜+브랜드 컬러 배지로 대체한다.

### Key Entities *(include if feature involves data)*

- **종목 표시 항목**: 화면에 나열되는 한 줄. 식별을 위한 종목 코드(심볼)와 표시명을 가지며, 이
  둘로부터 로고/폴백이 결정된다. 현재 일부 화면은 표시명만 넘기고 종목 코드를 함께 넘기지 않아
  로고가 적용되지 않는다.
- **로고 폴백 규칙**: 종목 코드·유형으로 표시할 이미지 후보와, 모두 실패했을 때의 이니셜·색을
  결정하는 기존 공통 규칙. 본 기능은 이 규칙을 바꾸지 않고, 누락된 화면이 이 규칙을 타도록 한다.
- **계좌**: 증권사 식별자와 표시명을 가진다. 증권사 식별자로 증권사 로고/브랜드 컬러를 결정한다.
  로고가 없으면 기존 이니셜+컬러 배지로 폴백한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 보유 목록에서 로고가 보이는 모든 종목이, 자산배분 화면과 거래 내역 화면에서도 동일한
  로고로 표시된다(같은 종목의 화면 간 로고 불일치 0건).
- **SC-002**: 종목이 없는 행·거래(현금·입출금·환전)에서 잘못된 로고가 표시되는 사례 0건.
- **SC-003**: 로고가 없는 종목에서 깨진 이미지나 빈 아이콘이 나타나는 사례 0건(항상 이니셜·색
  폴백으로 대체).
- **SC-004**: 사용자가 자산배분·거래 내역 화면에서 다른 화면으로 이동했을 때 같은 종목을 "다른
  종목처럼" 인지하는 일이 없도록, 종목 아이콘 표현이 화면 전체에서 일관된다.
- **SC-005**: 로고 에셋이 확보된 증권사 계좌는 계좌 화면에서 증권사 로고로 표시되고, 에셋이 없는
  계좌도 깨진 이미지 없이 이니셜+컬러로 안정적으로 표시된다(깨진 아이콘 0건).

## Assumptions

- "return allocation 페이지"는 자산배분(Allocation) 계열 화면을 가리키며, 수익률(Returns) 목록은
  이미 로고가 정상 표시되는 것으로 본다(필요 시 확인). 핵심 누락 화면은 거래 내역과 자산배분이다.
- 거래 내역의 종목 로고 배치는 명확화됨: 다른 화면과 동일한 공용 로고 표시를 그대로 재사용하고,
  거래 유형은 기존 텍스트 라벨로 구분한다(Clarifications 참조).
- 새로운 로고 데이터 소스나 외부 의존을 추가하지 않으며, 기존 공통 로고/폴백 규칙을 재사용한다.
- 종목 코드는 해당 화면 데이터에 이미 존재하거나 표시명으로부터 신뢰성 있게 얻을 수 있다고 본다.
- **증권사 로고는 현재 라이선스 부담을 이유로 의도적으로 이니셜+컬러로 대체해 온 영역이다(기존
  결정의 역전).** 소싱은 종목 로고와 동일한 기존 파이프라인을 따른다(셀프 호스팅 → 도메인 favicon
  폴백 → 이니셜+컬러). 셀프 호스팅 에셋이 없는 증권사는 favicon 또는 기존 배지로 자연스럽게
  폴백되므로, 별도 에셋 일괄 확보 없이도 점진 적용이 가능하다.
- 계좌의 증권사 식별자는 데이터에 이미 존재한다(계좌 생성 시 선택). 미지정 계좌도 허용된다.
