# Feature Specification: 종목 로고 이미지 · 지수 지표 표시 · 환율 상세

**Feature Branch**: `009-asset-logos-index-fx-detail`
**Created**: 2026-06-24
**Status**: Draft
**Input**: User description: "우리 화면에 있는 모든 종목 이미지를 기업, 운용사, 국가, 비트코인이미지 로 바꾸고 싶어 지금은 동그라미 안에 글씨가 써져있어. 그리고 코스피 지수 상세에서 PER등 지표 안뜨는거 이유 찾아보고, 환율 상세페이지 만들 계획세워줘"

## Summary

세 가지를 하나의 기능으로 묶는다.

1. **종목 로고** — 현재 모든 화면의 종목 아이콘은 "동그라미 안 글자"(이름 첫 글자) 폴백이다. 이를 자산 유형에 맞는 실제 이미지로 바꾼다: 기업(회사 로고), ETF(운용사 로고), 지수(국가/시장 이미지), 암호화폐(코인 이미지).
2. **지수 상세 지표** — 코스피(KOSPI) 지수 상세에서 PER 등 지표가 비어 보이는 원인을 규명하고, 데이터 출처가 있는 지표는 반드시 표시한다. 선행 PER(forward PER)은 예측값인 데다 국내·미국 지수 모두 신뢰할 출처가 없으므로 지표 목록에서 **제거**한다.
3. **환율 상세 페이지** — 현재 환율은 현금 탭의 목록 한 줄로만 보인다. 종목·지수 상세와 같은 패턴의 환율 상세 페이지를 신설한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 자산마다 알아볼 수 있는 이미지 (Priority: P1)

사용자가 보유 목록, 검색, 상세, 대시보드 등 어느 화면을 보든, 각 자산이 글자 동그라미가 아니라 그 자산을 바로 알아볼 수 있는 이미지로 표시된다. 기업은 회사 로고, ETF는 운용사 로고, 지수는 해당 국가/시장 이미지, 비트코인 등 암호화폐는 코인 이미지로 나타난다.

**Why this priority**: 앱 전반의 시각적 정체성과 신뢰감에 직접 영향을 주며, 거의 모든 화면(보유·계좌·배분·투시·수익률·검색·상세·대시보드)에 공통으로 쓰이는 요소다. 단독으로도 즉시 체감되는 가치를 준다.

**Independent Test**: 보유 목록과 검색 화면에서 대표적인 4개 유형(예: 삼성전자=기업, KODEX 200=운용사, 코스피=국가/지수, 비트코인=암호화폐)을 열어, 각 항목이 글자 동그라미가 아니라 해당 유형의 이미지로 보이는지 확인.

**Acceptance Scenarios**:

1. **Given** 보유 목록에 국내 대형 기업 종목이 있을 때, **When** 목록을 열면, **Then** 그 종목은 회사 로고 이미지로 표시된다.
2. **Given** ETF(예: KODEX, TIGER)를 보유 중일 때, **When** 목록·상세를 열면, **Then** 해당 ETF는 운용사 로고로 표시된다.
3. **Given** 관심목록에 지수(코스피/코스닥/S&P500 등)가 있을 때, **When** 화면을 열면, **Then** 해당 지수는 국가/시장을 나타내는 이미지(예: 국기)로 표시된다.
4. **Given** 비트코인 등 암호화폐 자산이 있을 때, **When** 화면을 열면, **Then** 코인 이미지로 표시된다.
5. **Given** 어떤 자산의 이미지를 찾을 수 없을 때, **When** 화면을 열면, **Then** 현재와 같은 글자 동그라미 폴백으로 표시되어 깨진 이미지가 보이지 않는다.
6. **Given** 동일한 자산이 여러 화면에 등장할 때, **When** 화면을 이동하면, **Then** 모든 화면에서 같은 이미지가 일관되게 표시된다.

### User Story 2 - 지수 상세에서 보이는 지표 채우기 (Priority: P2)

사용자가 코스피 등 지수 상세를 열면, 제공 가능한 가치평가 지표(PER, PBR, 배당수익률 등)가 실제 값으로 보인다. 선행 PER은 예측값이고 국내·미국 모두 신뢰할 출처가 없어 지표 목록에서 제거하며, 그 외 출처가 없는 지표가 있으면 빈칸이 아니라 "정보 없음"으로 안내한다.

**Why this priority**: 사용자가 명시적으로 "안 뜨는 이유"를 물은 버그성 항목이다. 로고만큼 광범위하진 않지만 신뢰도에 직접 영향을 준다.

**Independent Test**: 코스피 지수 상세를 열어 PER/PBR/배당수익률이 값으로 채워지는지, 그리고 값이 없는 지표가 빈 화면이 아니라 명시적 "정보 없음" 상태로 보이는지 확인.

**Acceptance Scenarios**:

1. **Given** 국내 지수(코스피/코스닥)에 대해 출처가 있는 지표(PER·PBR·배당수익률)가 존재할 때, **When** 지수 상세를 열면, **Then** 해당 지표들이 실제 값으로 표시된다.
2. **Given** 어떤 지수 상세든, **When** 지표 영역을 보면, **Then** 선행 PER 항목은 화면에 노출되지 않는다(국내·미국 모두).
3. **Given** 선행 PER을 제외한 특정 지표가 어떤 출처에서도 제공되지 않을 때, **When** 지수 상세를 열면, **Then** 그 항목은 "정보 없음"으로 표시되고, 전체 지표 영역이 통째로 비어 보이지 않는다.
4. **Given** 지표 데이터 캐시가 비어 있거나 동기화되지 않았을 때, **When** 지수 상세를 열면, **Then** 사용자에게 데이터가 아직 준비되지 않았음이 구분되어 보인다(영구 빈칸이 아니라 일시적 상태로).
5. **Given** 개별 종목 상세와 지수 상세를 비교할 때, **When** 둘 다 열면, **Then** 지수에서 제공되는 지표 항목 집합이 일관된 규칙(출처 유무)에 따라 결정되어 있다.

### User Story 3 - 환율 상세 페이지 (Priority: P3)

사용자가 현금/환율 화면의 환율 목록에서 특정 통화(예: USD/KRW)를 누르면, 종목·지수 상세와 동일한 형식의 환율 상세 페이지로 이동해 현재 환율, 변동, 추이 차트, 기간 고저 등을 볼 수 있다.

**Why this priority**: 신규 화면으로 가치가 명확하지만, 위 두 항목 없이도 앱은 동작한다. 기존 상세 페이지 패턴을 재사용하므로 마지막 순서가 적절하다.

**Independent Test**: 환율 목록에서 통화 한 줄을 눌러 상세 페이지로 진입하고, 현재 환율·변동·추이 차트가 표시되며 뒤로가기로 목록에 돌아오는지 확인.

**Acceptance Scenarios**:

1. **Given** 환율 목록에 통화 행이 있을 때, **When** 한 행을 누르면, **Then** 해당 통화의 환율 상세 페이지로 이동한다.
2. **Given** 환율 상세 페이지가 열렸을 때, **When** 화면을 보면, **Then** 현재 환율, 일간 변동(금액·%), 추이 차트, 기간 고저(예: 52주)가 표시된다.
3. **Given** 환율 상세 페이지에서 뒤로가기를 누를 때, **When** 동작하면, **Then** 환율 목록 화면으로 자연스럽게 복귀한다.
4. **Given** 추이 데이터를 일시적으로 가져올 수 없을 때, **When** 상세를 열면, **Then** 현재 환율은 보이고 차트 영역은 빈 화면이 아니라 데이터 없음/재시도 상태로 보인다.
5. **Given** 환율 상세 페이지가 다른 상세 페이지(종목·지수)와 함께 쓰일 때, **When** 화면 구조를 비교하면, **Then** 하단 탭바·뒤로가기·헤더 레이아웃이 동일한 패턴을 따른다.

---

### Edge Cases

- **로고 출처 불일치**: 같은 운용사의 여러 ETF가 모두 같은 운용사 로고로 보이는 것이 의도된 동작임을 사용자에게 혼란 없이 전달한다(개별 ETF 로고가 아님).
- **이미지 로딩 실패/지연**: 이미지가 늦게 오거나 실패하면 깨진 이미지가 아니라 즉시 글자 동그라미 폴백으로 대체된다.
- **신규/마이너 종목**: 로고 데이터가 없는 중소형주·해외 비주요 종목은 폴백으로 표시되며, 화면 레이아웃이 깨지지 않는다.
- **지수 지표 부분 결측**: (선행 PER 제외) 일부 지표만 있고 일부는 없을 때, 있는 것은 값으로 없는 것은 "정보 없음"으로 항목별로 구분된다.
- **환율 통화 다양성**: USD 외 통화(JPY·EUR 등) 상세 진입 시에도 동일한 구조가 적용되거나, 지원 범위가 명확히 한정된다.
- **다크/라이트 및 작은 사이즈**: 작은 아바타 크기(목록 행)에서도 로고가 알아볼 수 있게 보인다.

## Requirements *(mandatory)*

### Functional Requirements

**종목 로고 (User Story 1)**

- **FR-001**: 시스템은 모든 화면의 종목 아이콘을 자산 유형(기업/운용사/지수·국가/암호화폐)에 맞는 이미지로 표시해야 한다.
- **FR-002**: 시스템은 자산을 네 가지 유형으로 분류하고, 유형별로 적절한 이미지 종류를 선택해야 한다: 기업=회사 로고, ETF=운용사 로고, 지수=국가/시장 이미지, 암호화폐=코인 이미지.
- **FR-003**: 시스템은 이미지를 찾지 못한 자산에 대해 기존의 글자 동그라미 폴백을 유지해, 깨진 이미지가 노출되지 않도록 해야 한다.
- **FR-004**: 시스템은 동일 자산에 대해 모든 화면에서 동일한 이미지를 일관되게 보여야 한다.
- **FR-005**: 시스템은 이미지 로딩이 실패하거나 지연될 때 폴백으로 안전하게 전환해야 한다.
- **FR-006**: 로고 표시는 기존 아바타가 쓰이는 모든 위치(보유 목록, 계좌 상세, 배분, 투시, 수익률, 검색/관심목록, 종목 상세, 대시보드, 거래/온보딩 선택 등)에 적용되어야 한다.

**지수 상세 지표 (User Story 2)**

- **FR-007**: 시스템은 지수 상세에서 데이터 출처가 존재하는 가치평가 지표(최소 PER·PBR·배당수익률)를 실제 값으로 표시해야 한다.
- **FR-008**: 시스템은 선행 PER(forward PER)을 지수 상세 지표 목록에서 제거해야 한다(국내·미국 지수 모두 노출하지 않음).
- **FR-009**: 시스템은 선행 PER을 제외한 지표 중 어떤 출처에서도 제공되지 않는 지표를 빈칸이 아니라 "정보 없음"으로 명시해, 지표 영역이 통째로 비어 보이지 않게 해야 한다.
- **FR-010**: 시스템은 지표 데이터가 아직 동기화되지 않은 일시적 상태와, 본질적으로 데이터가 없는 영구 상태를 사용자에게 구분해 보여야 한다.
- **FR-011**: 시스템은 코스피 지수 상세에서 지표가 비어 보였던 원인을 해소해, 출처가 있는 지표가 실제로 화면에 나타나도록 해야 한다.

**환율 상세 (User Story 3)**

- **FR-012**: 시스템은 환율 목록의 각 통화 행에서 진입 가능한 환율 상세 페이지를 제공해야 한다.
- **FR-013**: 환율 상세 페이지는 현재 환율, 일간 변동(금액·%), 추이 차트, 기간 고저를 표시해야 한다.
- **FR-014**: 환율 상세 페이지는 기존 종목·지수 상세와 동일한 화면 패턴(하단 탭바·뒤로가기·헤더 레이아웃)을 따라야 한다.
- **FR-015**: 시스템은 추이 데이터를 일시적으로 가져오지 못할 때 현재 환율은 유지하고 차트 영역만 데이터 없음/재시도 상태로 표시해야 한다.
- **FR-016**: 시스템은 환율 상세에서 지원하는 통화 범위를 명확히 정의해야 한다(최소 USD/KRW 포함).

### Key Entities *(include if data involved)*

- **자산(Asset/Security)**: 화면에 표시되는 종목·ETF·지수·암호화폐. 식별 속성: 심볼, 이름, 자산 유형, 국가, 거래소, 통화. 이미지 선택의 기준이 된다.
- **로고/이미지 자원**: 자산 유형별 이미지(회사 로고·운용사 로고·국가 이미지·코인 이미지)와 폴백(글자 동그라미). 자산과 다대일로 연결된다(여러 ETF → 한 운용사 로고).
- **지수 지표(Index Metrics)**: 지수의 가치평가 지표 집합(PER·PBR·배당수익률·선행PER 등). 각 지표는 "값 있음 / 정보 없음 / 미동기화" 상태를 가진다.
- **환율(Exchange Rate)**: 통화쌍(예: USD/KRW)의 현재 값, 일간 변동, 추이 시계열, 기간 고저.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 보유 목록·검색·상세에서 네 가지 대표 유형(기업·운용사·지수·암호화폐) 각각이 글자 동그라미가 아닌 해당 유형 이미지로 표시된다(대표 자산 100% 확인).
- **SC-002**: 이미지를 찾지 못한 자산에서도 깨진 이미지가 0건이며, 항상 폴백이 표시된다.
- **SC-003**: 코스피 지수 상세에서 출처가 존재하는 지표(PER·PBR·배당수익률)가 값으로 표시되며, "지표가 통째로 비어 있는" 상태가 더 이상 발생하지 않는다.
- **SC-004**: 선행 PER 항목은 모든 지수 상세에서 노출되지 않으며(0건), 그 외 출처가 없는 지표는 빈칸이 아니라 "정보 없음"으로 100% 구분 표기된다.
- **SC-005**: 사용자가 환율 목록에서 통화를 눌러 체감상 즉시(3초 이내) 환율 상세에 도달하고, 현재 환율과 추이를 한 화면에서 확인할 수 있다.
- **SC-006**: 환율 상세의 화면 구조(탭바·뒤로가기·헤더)가 종목·지수 상세와 시각적으로 일관되어, 사용자가 별도 학습 없이 이용한다.

## Assumptions

- **로고 출처**: 새로운 유료/외부 의존을 추가하지 않는 범위에서, 기존 방식(심볼→도메인 매핑 기반 favicon, 운용사 약어·브랜드, 국기·코인 이미지)을 확장·정비하는 것을 우선한다. 모든 종목 100% 실제 로고 확보는 목표가 아니며, 미보유 시 폴백으로 처리한다.
- **운용사 단위 로고**: ETF는 개별 상품 로고가 아니라 운용사(KODEX=삼성, TIGER=미래에셋 등) 단위 로고로 표시하는 것을 기본으로 한다.
- **지수 지표 데이터 범위**: 국내 지수의 PER·PBR·배당수익률은 기존 KRX 통계 캐시에서 제공된다. 선행 PER은 예측값이고 국내·미국 모두 신뢰할 출처가 없어 지표 목록에서 제거한다. 그 외 출처가 없는 지표가 있으면 "정보 없음"으로 처리한다(없는 데이터를 만들어내지 않는다).
- **환율 추이 데이터**: 환율 시계열은 기존 시세 차트 인프라를 재사용해 통화쌍을 하나의 심볼처럼 다루는 방식을 가정한다. 새로운 외부 의존은 추가하지 않는다.
- **지원 통화**: 환율 상세는 현재 현금 탭에 노출되는 통화(USD 우선, 가능 시 JPY·EUR)를 대상으로 한다.
- **플랫폼**: 기존 모바일 우선 웹 UI와 동일한 디자인 기준(토스 느낌, 브랜드색)을 따른다.

## Dependencies

- 기존 아바타/브랜드색 표시 체계 및 자산 메타데이터(심볼·이름·유형·국가·거래소).
- 기존 KRX 지수 통계 캐시 및 지수 상세 화면.
- 기존 환율 조회 로직, 현금/환율 목록 화면, 시세 차트 인프라, 기존 상세 페이지 라우팅 패턴.
