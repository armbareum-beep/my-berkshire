# Feature Specification: 랭커 자산구성 도넛 + 보유 종목 공시

**Feature Branch**: `claude/ipo-button-founding-confirmation-r06fma` (main에서 재시작한 새 브랜치)
**Created**: 2026-07-05
**Status**: Implemented
**Input**: User description: "해당 랭커 종목도 볼 수 있었으면 좋을것 같은데" → "주식뿐 아니라 통 자산 나오게 해줘. 자산구성 탭에 있는 자료들처럼 도넛 그래프로 볼 수 있게 해줘"

## 개요

리더보드 프로필 시트는 유형별 구성 비중을 얇은 스택바로만 보여주고, 랭커가 어떤 종목을 들고 있는지는 볼 수 없었다(034·035 "보유 종목명 구조적 비공개" 불변식). 이 기능은 상장(IPO) 세계관을 공시(13F)까지 확장한다 — 상장한 회사는 보유 종목과 자산 구성을 시장에 공시한다.

**명시적 결정**:

1. **034 FR-009·035 불변식의 부분 개정** — 보유 종목명·심볼과 비중 %는 공개로 완화한다. **정확한 자산 금액·수량은 계속 구조적 비공개**(`ranking_scores`에 해당 컬럼이 없음). 자산은 "구간"만 공개되므로 비중 %로는 실제 금액이 역산되지 않는다.
2. **상장하면 자동 공개** — 별도 옵트인 토글 없음(사용자 결정). 상장 카드(IpoCard)의 공개 항목 고지에 "자산 구성·보유 종목과 비중(%)"을 명시해 상장 시점에 알린다.
3. **"통 자산" 구성** — 구성 비중(`composition`)에 실물자산(수기 평가: 부동산/토지/상가·수익형/비상장·지분/실물·수집/기타)을 합류시킨다. 분모 = 투자자산 + 현금 + 실물자산(보유 중만, 매도 제외). 부채는 자산 구성이 아니므로 미포함.
4. **도넛 그래프** — 프로필 시트의 구성 비중 스택바를 자산구성 탭과 같은 `Donut` 컴포넌트로 교체(범례 색 `donutColor(i)` 일치). 도넛은 %만으로 그린다(금액 없음).
5. **전 종목 공개** — 보유 종목은 상위 N이 아니라 전부, 비중 내림차순. 반올림 0%인 소액 종목도 목록에 유지("1% 미만" 표시). 비중 분모는 주식계좌(종목+현금) — 항목 합 = 100 − 현금 몫.
6. **전원 재동의 없음** — 035가 XIRR·자산 구간을 공개로 넓혔을 때와 같은 방식으로 기존 상장자에게도 적용한다. 기존 행은 다음 upsert(본인 `/ranking`·`/dashboard` 방문) 때부터 채워진다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 랭커의 자산 구성과 보유 종목을 본다 (Priority: P1)

리더보드에서 랭커를 탭하면 프로필 시트에 (기존 점수·연혁에 더해) 실물자산까지 포함한 자산 구성 도넛과 보유 종목 목록(종목명 + 비중 %)이 보인다.

**Why this priority**: 이 기능의 핵심 — 공시 열람 그 자체.

**Independent Test**: 상장 계정 A가 `/ranking` 방문(스냅샷 upsert) → 다른 계정 B가 리더보드에서 A의 시트를 열어 도넛(실물 라벨 포함)과 종목 목록 확인 → 어떤 화면에도 금액·수량이 없는지 확인.

**Acceptance Scenarios**:

1. **Given** 상장 계정의 최신 스냅샷, **When** 프로필 시트를 열면, **Then** "자산 구성" 도넛(유형 + 실물 라벨, %)과 "보유 종목" 목록(로고·종목명·비중 바·%)이 보인다.
2. **Given** 실물자산이 있는 상장 계정, **When** 스냅샷이 갱신되면, **Then** 도넛에 부동산 등 실물 라벨이 유형(주식→ETF→원자재→코인→현금) 뒤에 금액 내림차순으로 나타나고 합은 100%다.
3. **Given** 비중 0.4%짜리 소액 종목, **When** 보유 종목 목록을 보면, **Then** 목록에 존재하며 "1% 미만"으로 표시된다.
4. **Given** 038 이전에 저장된 행(`holdings` null), **When** 시트를 열면, **Then** 보유 종목 섹션만 생략되고 크래시 없다(구성 비중은 구 슬라이스 그대로 도넛으로 렌더).

### Edge Cases

- 시세 실패(`priceAvailable=false`) → `composition`·`holdings` 모두 null 저장, 섹션 생략(금액을 모르는데 %를 추정하지 않음 — 035와 동일).
- 스냅샷 신선도 — 표시값은 랭커 본인이 마지막으로 `/ranking`·`/dashboard`를 방문한 시점 기준(035 표시 컬럼과 동일한 수용된 한계).
- 상장폐지 → `ranking_scores` 행 삭제로 종목·구성도 함께 비노출(036 메커니즘 그대로).
- 실물자산이 없는 유저 → 도넛은 기존 유형 구성과 동일(회귀 없음). cap_rate 평가 자산은 `applyCapRateValuation` 반영 후 합산, 매도된 실물자산은 제외.
- 현금만 있고 종목이 없는 유저 → `holdings` null(공시할 종목 없음), 도넛은 현금 100%.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: `computeCompositionPct`는 옵션 `manual`(실물자산 종류별 ₩ 합산)을 받아 분모와 슬라이스에 합류시켜야 한다. 미전달 시 기존 동작과 비트 동일해야 한다(회귀 안전).
- **FR-002**: 신규 `ranking_scores.holdings` jsonb(v1)에 보유 종목 스냅샷을 저장해야 한다 — `{"v":1,"items":[{symbol,name,pct}]}`, 비중 내림차순, 반올림 0% 포함, 금액·수량 필드 없음. 시세 실패·종목 없음이면 null.
- **FR-003**: 스냅샷 쓰기는 두 upsert 경로(`/ranking` 동기, `/dashboard` after 백그라운드) 모두에서 동일하게 수행해야 한다(`upsertRankingScore` opts 확장).
- **FR-004**: 프로필 시트는 `composition`을 `Donut`(자산구성 탭과 동일 컴포넌트·팔레트)으로, `holdings`를 종목 목록(로고+이름+비중 바+%)으로 렌더해야 하며, 각 데이터가 null/빈 값이면 해당 섹션을 생략해야 한다.
- **FR-005**: `parseHoldingsV1`은 방어적으로 파싱해야 한다(`v≠1`·비배열 → null, 손상 항목 필터).
- **FR-006**: IpoCard 공개 항목 고지는 자산 구성·보유 종목 공개를 명시하고, "보유 종목명은 공개되지 않아요" 문구를 제거해야 한다(개정된 불변식: "정확한 금액·수량은 공개되지 않아요").

### Key Entities

- **`ranking_scores.holdings`** (jsonb, nullable, 신규): 표시 전용 — 점수 산정 무관. 마이그레이션 `20260705100000_ranking_scores_v4_holdings.sql`.
- **`CompositionV1`**: shape 불변(`{v:1, slices:[{label,pct}]}`) — 라벨 집합만 실물 종류로 확장(additive, 구버전 행·파서와 하위호환).
- **`HoldingsV1`** (신규, `src/lib/rankingHoldings.ts`): `{v:1, items:[{symbol,name,pct}]}`.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 상장 계정의 스냅샷 갱신 후, 타 계정 프로필 시트에서 그 계정의 전 보유 종목이 비중 내림차순으로 보인다.
- **SC-002**: 실물자산 보유 계정의 도넛 슬라이스 합이 100이고 실물 라벨이 포함된다.
- **SC-003**: `ranking_scores`의 어떤 컬럼에도 정확한 금액·수량이 존재하지 않는다(holdings·composition은 %와 라벨·심볼·이름만).
- **SC-004**: 038 이전 행(holdings null)의 프로필 시트가 크래시 없이 렌더된다.

## Assumptions

- 035의 표시 컬럼 패턴(점수 무관, upsert 시 스냅샷, 방문 기준 신선도)을 그대로 따른다. `SCORE_VERSION`·점수 공식 변경 없음.
- 공시는 상장의 일부라는 세계관(13F) — 별도 동의 UI 없이 상장 고지 문구로 알린다(사용자 결정).
- ETF·코인 등 종목 아닌 유형도 `positions`에 있으면 보유 종목 목록에 포함된다(종목 = 시세 피드 자산 전부).
