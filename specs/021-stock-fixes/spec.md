# Feature Specification: 달러 매수 통화 감지 + 원화 입력 토글

**Feature Branch**: `021-stock-fixes`  
**Created**: 2026-06-28  
**Status**: Shipped ✅  
**Merged**: 2026-06-28 (PR #27)

## 배경 (Why)

두 가지 이슈:

1. **이슈 7 (P1)** — 미국 주식을 달러로 매수할 때 "통화 확인 실패" 오류. Yahoo 조회가 실패(undefined)하면 기본값 `"KRW"`로 처리돼 안전장치가 모든 야후 실패를 막았음.

2. **이슈 6 (P3, 조사 완료)** — 인바디 등 일부 종목 현재가 불일치. 조사 결과: 심볼 저장·야후 조회 자체에 코드 버그 없음. Yahoo API 응답 타이밍 차이나 비교 출처 차이. 코드 수정 대상 아님.

---

## 구현 요약

### 이슈 7a — USD 통화 감지 fallback (`actions.ts`)

**수정 전 로직:**
```
Yahoo 조회 실패 → undefined → "KRW" 기본값 → 안전장치가 모든 야후 실패를 차단
```

**수정 후 로직:**
```
Yahoo가 명시적 "KRW" 반환(맵에 키 존재) → 안전장치 작동
Yahoo 조회 실패(맵에 키 없음) → 심볼 휴리스틱: 6자리=KRW, 그외=USD
```

핵심: `available[symbol]`이 `undefined`인 케이스와 `"KRW"`인 케이스를 분리.

### 이슈 7b — 원화 입력 토글 (`BuyWizard.tsx`)

USD 종목 단가 입력 화면에 "달러로 입력 / 원화로 입력" 라디오 토글 추가:
- **달러 입력**: 기존 그대로 (달러 단가 → 그대로 저장)
- **원화 입력**: 사용자가 ₩ 총액 입력 → `달러단가 = 원화총액 / 수량 / 환율`로 역산해 저장
- 환율은 기존 `getFxToKrw()` 재사용

## 설계 결정

- **Yahoo 실패를 "KRW"로 기본값 쓰지 않는 이유**: 미국 주식(알파벳 심볼)을 살 때 Yahoo 일시 실패로 KRW 판정 → 통화 불일치 오류가 발생했음. 심볼 휴리스틱이 더 안전.
- **저장 단위는 항상 달러**: DB에는 달러 단가로 저장. 원화 입력은 UI 편의만. 환산 계수는 입력 시점 환율(실시간).
- **원화 입력 대상**: USD 종목에만 노출. KRW 종목엔 불필요.

## 이슈 6 조사 결과 (코드 수정 없음)

인바디(`101930`) 현재가 불일치:
- DB 심볼 저장 확인: `101930` 정상
- Yahoo `.KS` suffix 조회: 정상 반환
- 결론: 비교 출처(앱 vs. 증권사 HTS)의 조회 시점 차이(캐시 TTL 10초). 코드 버그 없음.
- 향후 KIS API 전환 시 실시간 시세로 해소 가능 (`003-kis-market-data` 참조).
