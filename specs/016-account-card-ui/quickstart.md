# Quickstart: 계좌 카드 UI 개선 검증

신규 외부 인터페이스 없음(내부 UI 변경) → `contracts/` 생략. 수동 구동으로 검증한다.

## 사전 조건
- 분류가 서로 다른 계좌 2개 이상(예: 같은 증권사의 일반 + 연금저축/IRP), 그중 하나는 **평가액이 큰** 계좌.
- `npm run dev`로 앱 구동(또는 프로젝트 표준 실행).

## 검증 절차

### US1 — 거래 위저드 카드식 계좌 선택 (P1)
1. 거래 입력 → **매수** 시작 → "어느 계좌인가요?" 단계.
   - [ ] 계좌가 세로 카드 리스트로, 각 카드에 증권사 로고(또는 아바타) + 이름 + 분류 배지 표시.
   - [ ] 카드 탭 시 선택 표시(ring), "다음" 진행 가능.
2. 거래 입력 → **배당/매도/증자/인출/환전** 중 하나(비-BUY, TxnWizard) → 계좌 단계.
   - [ ] 동일한 카드식 UI.
   - [ ] 계좌 선택 후 종목/수량 단계로 정상 진행(선택 종목·수량 초기화 부수효과 유지 확인).

### US2 — accounts 페이지 오버플로 (P2)
3. accounts 페이지를 모바일 폭(예: 390px)으로 열기.
   - [ ] 큰 평가액 금액이 카드 안에 한 줄로 잘리지 않게 표시.
   - [ ] 이름이 긴 계좌는 이름이 `…`로 잘리고 분류·금액은 온전.

### US3 — 홈 보유 계좌 (P3)
4. 홈(대시보드) "보유 계좌" 섹션.
   - [ ] 각 계좌에 분류 라벨(`연금저축 · 자회사 N개` 등) 표시.
   - [ ] 큰 평가액에서도 금액이 카드 밖으로 넘치지 않음.

## 품질 게이트
- [ ] 변경 파일 `npx tsc --noEmit` 클린(`AccountOption.broker` 추가가 모든 사용처와 호환).
- [ ] 변경 파일 `npx eslint` 클린.
- [ ] 디자인 톤 회귀 없음(흰 카드·그림자, 신규 색면 없음 — 원칙 IV).
