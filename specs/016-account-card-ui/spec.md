# Feature Specification: 계좌 카드 UI 개선

**Feature Branch**: `016-account-card-ui`  
**Created**: 2026-06-26  
**Status**: Draft  
**Input**: User description: "어카운트 페이지 계좌 카드 금액이 카드 밖으로 나감. 거래(매수 등)에서 계좌를 고를 때 증권사/연금저축/IRP 분류가 안 보여 어떤 계좌인지 모름 — 알약식 말고 카드식으로. 홈 보유 계좌도 동일."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 거래 입력 시 계좌를 한눈에 구분 (Priority: P1)

사용자가 거래(매수·매도·배당·증자·인출·환전 등)를 기록할 때 "어느 계좌인가요?" 단계에서 계좌를 고른다. 현재는 계좌 **이름만** 작은 알약 버튼으로 떠서, 같은 증권사에 일반·연금저축·IRP를 여러 개 가진 사용자는 어떤 계좌인지 구분하지 못한다. 각 계좌를 증권사 로고·이름·분류(일반/ISA/연금저축/IRP/해외)가 보이는 카드로 제시해, 잘못된 계좌에 거래가 귀속되는 일을 막는다.

**Why this priority**: 잘못된 계좌 선택은 세금 계산·수익률·자산 귀속을 전부 틀어지게 하는 데이터 정합성 문제다. 모든 거래 종류의 진입점이라 영향 범위가 가장 넓다.

**Independent Test**: 거래 입력에서 매수와 비-매수(예: 배당) 각각 진입해 계좌 선택 화면이 카드식으로 뜨고, 각 카드에 분류가 표시되며, 선택이 거래에 정확히 귀속되는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 일반·연금저축·IRP 계좌를 가진 사용자가 매수를 시작했을 때, **When** "어느 계좌인가요?" 단계에 도달하면, **Then** 각 계좌가 증권사 로고·이름·분류 배지를 가진 카드로 세로 리스트로 표시된다.
2. **Given** 계좌 선택 화면에서, **When** 한 계좌 카드를 탭하면, **Then** 그 카드가 선택 상태로 표시되고 "다음"으로 진행 가능하다.
3. **Given** 매수가 아닌 거래(매도·배당·증자·인출·환전 등)를 시작했을 때, **When** 계좌 선택 단계에 도달하면, **Then** 동일한 카드식 UI가 표시되고 이후 종목/수량 단계로 정상 진행된다.

### User Story 2 - accounts 페이지 금액이 카드 안에 보임 (Priority: P2)

사용자가 계좌 관리(accounts) 페이지를 볼 때, 평가액이 큰 계좌의 금액 텍스트가 카드 경계를 벗어나 잘리거나 화면 밖으로 삐져나간다. 금액은 항상 카드 안에 한 줄로 보이고, 계좌 이름이 길면 이름 쪽이 줄임표로 처리되어야 한다.

**Why this priority**: 시각적 깨짐이라 데이터 손상은 아니지만 신뢰감을 떨어뜨린다. 평가액이 클수록 더 자주 발생한다.

**Independent Test**: 평가액이 큰 계좌가 있는 상태에서 accounts 페이지를 좁은(모바일) 폭으로 열어 금액이 카드 안에 들어오는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 평가액이 매우 큰 계좌가 있을 때, **When** accounts 페이지를 모바일 폭에서 보면, **Then** 금액이 카드 안에 한 줄로 잘리지 않고 표시된다.
2. **Given** 이름이 매우 긴 계좌가 있을 때, **When** 카드를 보면, **Then** 이름은 줄임표(…)로 잘리고 금액과 분류는 온전히 보인다.

---

### User Story 3 - 홈 보유 계좌에 분류 표시 + 오버플로 방지 (Priority: P3)

사용자가 홈(대시보드) "보유 계좌" 섹션을 볼 때, 계좌 분류(증권사/연금저축/IRP)가 표시되지 않아 accounts 페이지와 정보가 어긋난다. 또한 금액 오버플로 방어가 약하다. 홈에서도 분류를 표시하고 금액이 카드 안에 머물게 한다.

**Why this priority**: 일관성·완성도 개선. 핵심 기능은 아니지만 같은 패턴을 앱 전체에 통일시킨다.

**Independent Test**: 홈 보유 계좌 섹션에서 각 계좌에 분류 라벨이 보이고, 큰 평가액에서도 금액이 카드 안에 머무는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 분류가 서로 다른 계좌들이 있을 때, **When** 홈 보유 계좌 섹션을 보면, **Then** 각 계좌에 분류 라벨이 "자회사 N개"와 함께 표시된다.
2. **Given** 평가액이 큰 계좌가 있을 때, **When** 홈에서 보면, **Then** 금액이 카드 밖으로 넘치지 않는다.

### Edge Cases

- 증권사가 지정되지 않은 계좌: 로고 대신 이름 글자 기반 아바타로 폴백(기존 동작 유지).
- 계좌가 1개뿐이거나 매우 많은(예: 10개+) 경우: 세로 리스트가 스크롤로 모두 보여야 한다.
- 금액·이름이 둘 다 길 때: 분류·금액 우선, 이름은 줄임표 처리.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 모든 거래 종류(매수·매도·배당·증자·인출·환전 등)의 계좌 선택 단계는 계좌를 카드식 세로 리스트로 표시해야 한다.
- **FR-002**: 각 계좌 카드는 증권사 로고(미지정 시 이름 아바타), 계좌 이름, 분류 배지(일반/ISA/연금저축/IRP/해외)를 표시해야 한다.
- **FR-003**: 사용자는 계좌 카드를 탭해 선택할 수 있어야 하며, 선택된 카드는 시각적으로 구분되어야 한다.
- **FR-004**: 계좌 선택 시 기존의 부수 동작(선택 종목·수량 초기화 등)이 그대로 유지되어 이후 단계가 정상 동작해야 한다.
- **FR-005**: accounts 페이지의 계좌 카드에서 금액은 카드 경계 안에 한 줄로 표시되어야 하며, 공간이 부족하면 계좌 이름이 줄임표로 잘려야 한다.
- **FR-006**: 홈 보유 계좌 섹션의 각 계좌에 분류 라벨을 표시해야 한다.
- **FR-007**: 홈 보유 계좌 섹션에서 금액은 카드 경계 안에 머물러야 한다.
- **FR-008**: 카드에 표시되는 분류·로고 정보는 accounts 페이지와 동일한 출처·표기를 사용해 일관성을 유지해야 한다.

### Key Entities *(include if feature involves data)*

- **계좌(Account)**: 사용자의 거래·보유가 귀속되는 단위. 이름, 분류(일반/ISA/연금저축/IRP/해외), 증권사를 가진다. 이 기능은 새 데이터를 만들지 않고 기존 계좌의 분류·증권사 속성을 화면에 노출한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 같은 증권사에 분류가 다른 계좌를 2개 이상 가진 사용자가 거래 입력 시 화면만 보고 의도한 계좌를 100% 정확히 선택할 수 있다(분류가 항상 보이므로).
- **SC-002**: accounts 페이지·홈 보유 계좌에서 어떤 평가액 크기에서도 금액 텍스트가 카드 경계를 벗어나지 않는다(모바일 최소 폭 기준).
- **SC-003**: 계좌 분류·로고 표기가 accounts 페이지·홈·거래 선택 세 화면에서 동일하다(불일치 0건).

## Assumptions

- 계좌 분류·증권사 정보는 이미 데이터에 존재하며(계좌 관리에서 설정), 이 기능은 표시만 추가한다.
- 거래 선택 카드는 정보 과밀을 피하기 위해 평가액·수익률은 표시하지 않는다(로고+이름+분류만). 평가액 표시는 범위 밖.
- 레이아웃은 세로 리스트(한 줄 한 계좌)로 하며, accounts 페이지 행과 동일한 톤을 따른다.
- 기존 증권사 로고/아바타·분류 라벨 표시 컴포넌트를 재사용한다(신규 디자인 자산 없음).
