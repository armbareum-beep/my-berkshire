# Quickstart: 토스식 계좌 만들기 검증

## 사전
- 로그인 + 온보딩 완료된 holding 1개(계좌 ≥ 0). 개발 서버 구동(`run` 스킬 또는 프로젝트 표준 dev 명령).

## US2 — CTA 진입 (목록부터, 누르면 폼)
1. `/accounts` 진입.
2. **기대**: 계좌 목록(+수수료 카드)과 하단 "계좌 만들기" 버튼만 보인다. 계좌 추가 입력 폼(이름·종류·증권사)이
   펼쳐져 있지 **않다**(SC-003).
3. "계좌 만들기" 클릭 → **기대**: 폼이 나타난다(이름 입력 + 종류 피커 + 증권사 [+ 컴퍼니 2개↑면 주인 select]).
4. 닫기(또는 추가 완료) → **기대**: 폼이 사라지고 목록 화면으로 복귀(SC-004).
5. 계좌가 0개인 상태에서도 "계좌 만들기" 버튼이 보이는지 확인(FR-008).

## US1 — 종류 피커 (절세 설명 카드)
1. 폼을 연 뒤 종류 선택 영역 확인.
2. **기대**: 일반·ISA·연금저축·IRP·해외 5개가 각각 **아이콘 + 이름 + 한 줄 절세 설명** 카드로 보인다
   (알약 버튼 아님). 예: ISA "배당 비과세 · 연 2,000만원 납입한도"(SC-001).
3. 아이콘이 5종 모두 빈칸·깨짐 없이 라인 아이콘으로 렌더되는지 확인(FR-001, EmojiIcon 폴백 텍스트로 안 떨어짐).
4. "ISA" 카드 탭 → **기대**: 카드가 선택 강조 + 체크 표시. 다른 카드 탭 시 선택이 이동(단일 선택).
5. 계좌명 입력 후 "계좌 추가" → **기대**: 토스트 성공, 목록에 선택한 종류로 새 계좌 표시, 폼 접힘.

## 회귀 (불변 확인)
- 추가된 계좌의 종류·이름·증권사·수수료가 기존과 동일하게 저장·표시되는지(SC-005, FR-009).
- `AccountRow` 목록·수수료 랭킹 카드 표시가 기존과 동일한지.

## 품질 게이트
- `npx tsc --noEmit` 클린(신규 상수·컴포넌트 타입 통과).
- `npx eslint` (변경 파일) 클린.
- (선택) 정적 검증 테스트: 모든 `AccountType`이 `ACCOUNT_TYPE_DESCRIPTION`·`ACCOUNT_TYPE_EMOJI` 키를 가지며
  이모지가 EmojiIcon MAP에 존재하는지 1건.
