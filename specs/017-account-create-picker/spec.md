# Feature Specification: 토스식 계좌 만들기 — 절세 혜택을 드러내는 종류 피커 + CTA 진입

**Feature Branch**: `017-account-create-picker`  
**Created**: 2026-06-26  
**Status**: Draft  
**Input**: User description: "시작해줘. 그리고 어카운트 페이지에 계좌추가 처음부터 나오는게 아니라 계좌만들기 누르면 뜨게해줘"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 절세 혜택을 보고 계좌 종류를 고른다 (Priority: P1)

사용자가 새 계좌를 만들 때, 계좌 종류(일반·ISA·연금저축·IRP·해외)를 단순한 이름 버튼이 아니라
**아이콘 + 종류 이름 + 한 줄 절세 설명**이 담긴 카드 목록에서 고른다. 각 카드가 "이 계좌를 쓰면 뭐가
유리한지"(예: ISA "비과세 · 연 2,000만원 한도", 연금저축 "세액공제 13.2% · 연 600만원 한도")를
바로 보여줘, 세금 지식이 없어도 자기 상황에 맞는 계좌를 고를 수 있다.

**Why this priority**: 사용자가 직접 토스 화면을 레퍼런스로 지목한 핵심. 시스템은 이미 계좌 종류별
절세 규칙(비과세 한도·세액공제율·과세이연)을 알고 있는데 화면에는 종류 이름만 노출돼 그 가치가 묻혀 있다.
설명을 카드로 드러내는 것만으로 "왜 이 계좌인가"를 사용자가 스스로 판단하게 해, 독립적으로 가치를 전달한다.

**Independent Test**: 계좌 만들기 화면을 열어 다섯 종류가 각각 아이콘과 한 줄 절세 설명을 단 카드로
표시되는지, 한 카드를 탭하면 선택 표시(강조·체크)가 뜨고 그 종류로 계좌가 생성되는지 확인.

**Acceptance Scenarios**:

1. **Given** 계좌 만들기 흐름에 진입한 상태, **When** 계좌 종류 목록을 본다, **Then** 일반·ISA·연금저축·IRP·해외
   각각이 아이콘 + 이름 + 한 줄 절세 설명 카드로 표시된다(이름만 있는 알약 버튼이 아니다).
2. **Given** 계좌 종류 카드 목록, **When** "ISA" 카드를 탭한다, **Then** 해당 카드가 선택 상태로 강조되고
   비과세·납입한도 설명이 함께 보인다.
3. **Given** 종류를 선택하고 계좌명을 입력한 상태, **When** 만들기를 확정한다, **Then** 선택한 종류로
   계좌가 생성되고 목록에 나타나며 성공 안내가 표시된다(기존 생성 동작과 동일 결과).
4. **Given** 다섯 종류 카드, **When** 화면을 본다, **Then** 모든 종류에 아이콘이 빈칸·깨짐 없이 렌더된다.

---

### User Story 2 - 어카운트 페이지는 목록부터, 만들기는 눌러야 뜬다 (Priority: P2)

어카운트(계좌) 페이지에 들어가면 처음부터 계좌 추가 입력 폼이 펼쳐져 있지 않고, **내 계좌 목록**이 먼저 보인다.
새 계좌가 필요할 때만 "계좌 만들기"를 눌러 생성 흐름(이름 입력 + 종류 피커 + 증권사)을 띄운다. 평소엔 화면이
계좌 현황에 집중되고, 생성 폼은 의도가 있을 때만 등장한다.

**Why this priority**: 사용자가 명시적으로 요청한 변경. 토스도 계좌 목록 + "새 계좌 만들기" CTA 구조다.
P1(피커)이 없어도 폼을 CTA 뒤로 숨기는 것만으로 화면 정돈 가치가 있어 독립적으로 테스트·배포 가능하다.

**Independent Test**: 어카운트 페이지 진입 시 입력 폼이 보이지 않고 계좌 목록과 "계좌 만들기" 버튼만 보이는지,
버튼을 눌러야 생성 흐름이 나타나고, 생성 완료(또는 취소) 후 다시 목록 화면으로 돌아오는지 확인.

**Acceptance Scenarios**:

1. **Given** 계좌가 1개 이상 있는 사용자가 어카운트 페이지에 진입, **When** 화면을 본다, **Then** 계좌 목록과
   "계좌 만들기" 버튼이 보이고 계좌 추가 입력 폼(이름·종류·증권사)은 펼쳐져 있지 않다.
2. **Given** 어카운트 페이지, **When** "계좌 만들기"를 누른다, **Then** 계좌 생성 흐름(이름 입력 + 종류 피커 +
   증권사 선택)이 나타난다.
3. **Given** 계좌 생성 흐름이 열린 상태, **When** 계좌를 추가하거나 흐름을 닫는다, **Then** 흐름이 사라지고
   계좌 목록 화면으로 돌아오며, 추가한 경우 목록에 새 계좌가 보인다.

---

### Edge Cases

- **계좌가 0개인 경우**: 목록이 비어 있을 때도 "계좌 만들기" 진입점은 항상 보여 사용자가 첫 계좌를 만들 수 있어야 한다(빈 상태 안내 + CTA).
- **계좌명 미입력**: 종류를 골라도 계좌명이 비어 있으면 생성 확정을 막고 안내한다(기존 검증 유지).
- **생성 흐름 도중 이탈**: 닫기·취소 시 입력값은 폐기하고 목록으로 복귀한다(부분 저장 없음).
- **종류 설명 문구 출처**: 카드의 절세 설명은 시스템이 보유한 계좌 종류별 세금 규칙과 일치해야 한다 — 화면 문구와 실제 적용 세제가 어긋나지 않는다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 계좌 종류 선택을 **아이콘 + 종류 이름 + 한 줄 절세 설명**을 담은 카드 목록으로 제시해야 한다(이름만 있는 단순 버튼 금지).
- **FR-002**: 시스템은 각 계좌 종류(일반·ISA·연금저축·IRP·해외)에 대해 해당 종류의 핵심 세제 특징을 한 줄로 표시해야 한다(예: 비과세 한도, 세액공제율, 과세 여부).
- **FR-003**: 종류 설명 문구는 시스템이 이미 보유한 계좌 종류별 세금·세액공제 규칙과 일치해야 한다(별도 수치를 새로 정의해 어긋나게 하지 않는다).
- **FR-004**: 사용자는 카드를 탭해 한 종류를 선택할 수 있어야 하고, 선택된 카드는 시각적으로 강조(체크 등)되어야 한다.
- **FR-005**: 어카운트 페이지는 진입 시 계좌 추가 입력 폼을 펼쳐 두지 않고 **계좌 목록과 "계좌 만들기" 진입점**을 먼저 보여줘야 한다.
- **FR-006**: 사용자가 "계좌 만들기"를 누르면 계좌 생성 흐름(계좌명 입력 + 종류 피커 + 증권사 선택)이 나타나야 한다.
- **FR-007**: 계좌 생성을 완료하거나 흐름을 닫으면 생성 흐름이 사라지고 계좌 목록 화면으로 돌아와야 하며, 추가한 계좌는 목록에 반영되어야 한다.
- **FR-008**: 계좌가 0개여도 "계좌 만들기" 진입점은 항상 접근 가능해야 한다.
- **FR-009**: 계좌 생성 결과(저장되는 종류·이름·증권사·수수료)는 기존 동작과 동일해야 한다 — 본 기능은 선택 UI와 진입 방식만 바꾸고 저장 데이터·계산 로직을 변경하지 않는다.

### Key Entities *(include if data involved)*

- **계좌 종류(기존)**: 일반·ISA·연금저축·IRP·해외. 각 종류는 표시용 이름과 세제 규칙(배당 과세 여부, 비과세/공제 한도, 세액공제율)을 가진다 — 카드의 아이콘·설명·선택값의 입력.
- **계좌(기존)**: 이름·종류·증권사·수수료율을 가진 생성 대상. 본 기능은 생성 흐름의 진입·선택 방식만 바꾼다(스키마 불변).

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 계좌 만들기 화면에서 다섯 종류 모두가 아이콘·이름·한 줄 절세 설명을 갖춘 카드로 표시된다(설명 없는 종류 0건).
- **SC-002**: 사용자는 세금 용어를 따로 찾아보지 않고도 각 카드의 한 줄 설명만으로 종류 간 절세 차이를 구분할 수 있다.
- **SC-003**: 어카운트 페이지 진입 시 계좌 추가 입력 폼이 펼쳐진 채 보이지 않는다(첫 화면에 폼 0건), 대신 목록과 "계좌 만들기" 진입점이 보인다.
- **SC-004**: "계좌 만들기"를 눌렀을 때 생성 흐름이 나타나고, 완료·취소 후 목록으로 복귀하는 왕복이 끊김 없이 동작한다.
- **SC-005**: 본 기능 도입 전후로 생성된 계좌의 저장 데이터(종류·이름·증권사·수수료)와 기존 계좌 표시·계산이 동일하다(회귀 0건).

## Assumptions

- 계좌 종류 집합과 각 종류의 세제 규칙은 기존 시스템 정의를 그대로 사용한다(신규 종류·세율 추가 없음).
- 카드의 한 줄 절세 설명 문구는 사용자가 이해하기 쉬운 요약 수준이며, 정밀 세법(소득구간별 공제율 등)은 본 기능 범위 밖이다.
- 계좌 생성에 필요한 입력(계좌명·증권사·수수료율)과 검증·저장 동작은 기존 흐름을 재사용한다.
- 생성 흐름을 화면 안에 펼치는 형태로 띄울지, 별도 시트/오버레이로 띄울지는 구현 단계(`/speckit.plan`)에서 기존 UI 패턴에 맞춰 결정한다 — 본 스펙은 "처음엔 숨기고 눌러야 뜬다"는 동작만 규정한다.
- 부동산·대체·사업 등 "사업부" 종류로의 피커 확장은 본 기능 범위 밖이다(추후 별도 기능).
