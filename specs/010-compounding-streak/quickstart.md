# Quickstart: 복리 무중단 지표 수동 검증

구현 후 아래 시나리오로 동작을 확인한다. (참조: spec.md Acceptance Scenarios / Success Criteria)

## 단위테스트 (자동, `compoundingStreak.test.ts`)

Vitest로 순수 함수 검증:

1. **무중단 기본**: DEPOSIT 1건(23개월 전), 인출 없음 → `months = 23`, `unit='month'`, `startDate`=그 투입일.
2. **첫날**: DEPOSIT 오늘 → `unit='day'`, `days=0`(또는 1), 표시 "N일".
3. **현금 비중 무관**: DEPOSIT + 대량 SELL(현금 90%) → 무중단 기간 **변화 없음**.
4. **소비성 인출 끊김**: DEPOSIT(12개월 전) + WITHDRAWAL(6개월 전) → `startDate`=인출일, `months ≈ 6`.
5. **배당 인출 무시**: DEPOSIT + DIVIDEND 인출 → 끊김 없음.
6. **추가 투입 보너스**: 최근 30일 내 DEPOSIT → `bonusRecentDeposit=true`.
7. **빈 장부**: 흐름 없음 → `isEmpty=true`.
8. **계좌 간 이동/매매**: BUY/SELL/EXCHANGE만 → 끊김 없음.
9. **같은 날 투입+인출**: 그날 순흐름 음수면 끊김, 양수면 유지.

품질 게이트: `npx tsc --noEmit` · `npx eslint` 클린.

## 수동 검증 (실제 앱)

`run`/`verify` 스킬로 구동 후:

1. **히어로 한 줄**: 대시보드 최상단에 "복리 무중단 N개월"이 평가금액과 함께 보인다. (SC-001)
2. **현금 불변**: 종목 일부 매도해 현금 비중을 올린다 → 무중단 기간이 **그대로**인지 확인. (SC-004)
3. **시세 불변**: 장중 시세 변동에도 무중단 기간 표시 불변. (SC-004)
4. **소비성 인출**: WITHDRAWAL 이벤트를 과거 날짜로 입력 → 무중단 기간이 그 시점 이후로 재계산. (SC-005)
5. **보너스**: 최근 DEPOSIT 추가 → 🔥 표시 등장.
6. **빈 장부**: 신규/빈 장부 → 죄책감 없는 중립 빈 상태(임의 시작일 없음).
7. **비율 단일화**: `/returns` 등에서 CAGR 행이 사라지고 "연 수익률"(XIRR)만 보인다. 한 화면에 백분율 2개 동시 노출 없음. (SC-003)
8. **결산 상세**: 분기 결산에서 시작일·추가 투입 이력·(있다면) 끊김 이력이 사실과 일치.

## 회귀 확인

- 기존 대시보드/결산/`/returns` 렌더·계산 불변(XIRR·수익률 숫자 변화 없음, CAGR만 화면에서 제거).
- 외화 보유 장부에서 ₩환산 결과 불변.
